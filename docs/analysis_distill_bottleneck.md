## 二、瓶颈根因分析

### 2.1 核心发现：79.52% 天花板

三次独立实验（Step 2 Round 2、Step 2 Round 3 两阶段、Step 3 多票融合）在 α=0.9 / seed=42 条件下均收敛于 **79.52%**，但无法超越单教师 DeepSeek 的 **81.93%**（seed=11 峰值）。

### 2.2 根因一：教师太强 → 软标签退化为 one-hot

融合后软标签的信息熵分析（672 条训练样本）：

| 熵区间 | 含义 | 样本数 | 比例 |
|--------|------|--------|------|
| < 0.1 | 近 one-hot (max prob > 0.95) | 554 | **82.4%** |
| 0.1 – 0.3 | 低熵 | 26 | 3.9% |
| 0.3 – 0.5 | 中等 | 10 | 1.5% |
| > 0.5 | 高熵（有信息量） | 82 | 12.2% |

- 平均熵 = **0.097**（理论最大 ln(5) ≈ 1.609）
- 融合答案与 GT 一致率 = **97.2%**

> **解释**：当 Doubao（98.8%）和 DeepSeek（87.95%）都答对时，融合分布的 dark knowledge 为零——student 只是在学习换一种方式表示 one-hot 标签。蒸馏的核心价值（传递教师对干扰项的排序信息）在此场景下不存在。

### 2.3 根因二：测试集太小，统计方差淹没方法差异

83 个测试样本，每对/错 1 题产生 **±1.20%** 的准确率变化。

| 配置 | seed=42 | seed=8 | seed=11 | 跨 seed 振幅 |
|------|---------|--------|---------|------------|
| Step 2 α=0.1 | — | 81.93% | 74.70% | **7.23 pp** |
| Step 3 α=0.9 | 79.52% | 75.90% | 74.70% | **4.82 pp** |

81.93% 与 74.70% 之间仅差 **6 道题**。在这个数据规模下，seed 引入的初始化 / dropout / 数据 shuffle 随机性产生的方差远大于任何蒸馏技术改进。

> **推论**：81.93% 更可能是某个 seed 的幸运峰值，而非可复现的方法优势。

### 2.4 根因三：α=0.9 最优 = 蒸馏信号几乎是噪声

在所有实验中，α=0.9（90% 用 GT loss，仅 10% 用蒸馏 loss）始终最优。当 α 降低（加大蒸馏权重），测试精度反而下降：

$$\mathcal{L} = \alpha \cdot \mathcal{L}_{GT} + (1 - \alpha) \cdot \mathcal{L}_{KD}$$

| α 值 | 蒸馏占比 | Test Acc |
|------|---------|----------|
| 0.9 | 10% | **79.52%** |
| 0.7 | 30% | 75.90% |
| 0.5 | 50% | 74.70% |
| 0.1 | 90% | 74.70%–81.93% (高方差) |

> **解释**：97% 的样本教师和 GT 完全一致，蒸馏 loss ≈ 0；剩余 3% 教师答错的样本上，蒸馏信号是负面的——学生在学习错误答案。因此蒸馏权重越大，引入的噪声越多。

### 2.5 根因四：自一致性过滤无增益

三种过滤策略（无过滤 / 默认 / 严格）结果完全相同（79.52%），因为：
- Doubao 平均自一致性 = 0.98（657/672 高一致）
- DeepSeek 平均自一致性 = 0.98（617/672 高一致）
- 仅 2 条样本回退到 GT

> **解释**：过滤的前提假设（存在大量低质预测需要剔除）在本数据集上不成立。

### 2.6 根因五：MCQ 任务的结构性局限

传统知识蒸馏的 dark knowledge 来自类之间的语义相似性（例如 ImageNet 中"猫"与"狗"比"猫"与"车"更相似）。但 MCQ 的 5 个选项之间没有固定的语义距离关系——一道题的选项 A-E 与另一道题的 A-E 语义完全不同。所以软标签在 MCQ 中仅编码"教师是否犹豫"，而非"选项间的知识结构"。

## 三、实验教训总结

1. **白盒蒸馏对 MCQ 无意义**：152K-dim logit KL 只有 5 个选项相关，信噪比极低（Module 01 → 73.49%，低于 GT 基线）
2. **弱教师会带来负蒸馏**：Qwen-14B (78.1%) 作为教师训练 7B 学生 → 75.90%，低于纯 GT SFT (77.11%)
3. **自蒸馏无意义**：Qwen-7B (71.4%) 教自己只会放大错误
4. **多教师融合不优于最强单教师**：融合取"平均"会稀释最强教师的优势
5. **小测试集下不宜追求单点峰值**：应关注多次实验的均值和标准差

## 四、改进方案

### 方案 A：多 Seed 集成推理（消除方差）

**目标**：不改变训练方法，通过推理阶段集成消除 seed 方差，获得可靠的性能评估。

**方法**：
1. 使用最佳配置（α=0.9, lr=1e-4, rank=16, 6 epochs）
2. 使用 7 个不同 seed（7, 8, 11, 42, 55, 77, 123）分别训练
3. 对每个模型在测试集上推理
4. 采用多数投票 (majority vote) 产生最终预测

**预期**：
- 消除 ±5pp 的单 seed 方差
- 稳定在 78%–80%
- 提供可靠的性能区间报告

**成本**：7 次 LoRA 训练（每次约 3 分钟）+ 7 次推理

#### 方案 A 实验结果

| Seed | Best Val (%) | Test Acc (%) |
|------|-------------|-------------|
| 7 | 74.32 | 75.90 |
| 8 | 78.38 | 75.90 |
| 11 | 77.03 | 74.70 |
| **42** | **75.68** | **79.52** |
| 55 | 75.68 | 78.31 |
| 77 | 77.03 | 74.70 |
| 123 | 74.32 | 71.08 |
| **均值** | **76.06** | **75.73** |
| **标准差** | **1.50** | **2.82** |

**7-Seed 多数投票集成：79.52% (66/83)**

**结论**：
- 集成结果 = 最佳单种子 (seed=42)，没有进一步提升
- 7 个 seed 的 test 标准差为 2.82pp（71.08%–79.52%），证实方差问题严重
- Val 最佳 epoch 与 test 表现无稳定相关性（seed=8 val 最高 78.38% 但 test 仅 75.90%）
- **多数投票定锚在中位水平，无法利用少数"幸运" seed 的高分**

**投票分析**：
- 71/83 题（85.5%）全票一致 → 模型在绝大部分题上行为一致
- 17 题错误中 **12 题是 7/7 全票一致错误** → 系统性错误，集成无法修正
- 仅 5 题存在投票分歧 → 集成只能修正极少数边界样本
- 说明瓶颈不在随机性，而在模型的知识边界——需要新的知识注入方式（→ 方案 B）

### 方案 B：推理链蒸馏 (Rationale Distillation)

**目标**：改变蒸馏范式——不再蒸馏 5-dim 答案分布，而是蒸馏教师的推理过程。

**动机**：当前蒸馏方案仅传递"选什么"，但教师（尤其是 Doubao 98.8%）的真正价值在于"为什么选"。Chain-of-Thought 推理链包含：
- 对各选项的排除逻辑
- 关键医学知识点
- 解题方法论

**方法**：
1. 用 Doubao API 对 672 条训练题生成 CoT 推理链
2. 构建 SFT 数据：Question → 推理过程 + Answer
3. 训练 Qwen-7B 学习"像教师一样思考"
4. 推理时让学生先生成推理再给答案

**不需要新的题目数据** — 使用现有 672 条训练集，只是将标签从"答案字母"升级为"推理过程+答案"。需要 API 调用 Doubao 为每题生成推理链。

**预期**：
- 将教师知识从 5-dim 升维到自然语言序列
- 推理链相当于"数据扩增"——同一道题承载了更丰富的监督信号
- 可能突破 79.52% 天花板

**实际结果**：

| 指标 | 值 |
|------|-----|
| CoT 生成 | 672 题全部处理，657 条 GT 匹配（97.8%），15 条 CoT 推导答案 ≠ GT 被剔除 |
| 训练配置 | lr=2e-4, 6 epoch, seed=42, max_length=2048 |
| 最佳 Val Acc | **74.32%** (epoch 1) |
| Val Acc 趋势 | epoch 1→6: 74.32% → 72.97% → 72.97% → 72.97% → 71.62% → 71.62% (持续下降) |
| **Test Acc** | **73.49%** (61/83) |

**结果分析**：

方案 B **未能突破天花板**，反而大幅低于所有主要方法：
- 低于纯 GT SFT 基线 77.11% → **-3.62pp**
- 低于最佳蒸馏 79.52% → **-6.03pp**
- 与白盒 KL（已废弃）持平 73.49%

**失败原因分析**：

1. **序列长度膨胀**：CoT 推理链将训练目标从 ~5 token（答案字母）扩展到 ~200-400 token，max_length=2048 截断率高，梯度信号被"推理噪声"稀释
2. **推理-答案不对齐**：评估时使用短答案模式（max_new_tokens=4），但训练时学的是长推理模式，推理/评估 prompt 不一致导致分布偏移
3. **7B 模型推理能力不足**：小模型学不会教师的推理模式，只学到了表面的语言风格，关键推理步骤无法内化
4. **Val 持续下降 = 过拟合推理模式**：epoch 1 即最佳，后续 epoch 加剧了推理模板的过拟合
5. **数据量不够支撑推理学习**：657 条 CoT 可能不足以让 7B 模型学会牙科推理范式

## 八、Plan C：升级学生模型容量（Module 13）

### 8.1 实验设置

**假设**：7B 天花板源于模型容量不足。将学生从 Qwen2.5-7B 升级为 Qwen2.5-14B-Instruct，复用 DeepSeek-V3 教师标签。

| 参数 | Stage 1 (Choice-Head) | Stage 2 (GT SFT) |
|------|----------------------|-------------------|
| 学生 | Qwen2.5-14B-Instruct | 同左 |
| 教师 | DeepSeek-V3 (87.95%) | 无 (纯 GT) |
| α | 0.35 | 0.0 |
| 训练轮数 | 1 epoch | 5 epochs |
| LoRA | rank=16, alpha=32 | 继承 Stage 1 |
| Seeds | 42, 11, 8 | 同左 |

### 8.2 完整结果

| Seed | Stage 1 Test | Stage 2 Test | Stage2 vs Stage1 |
|------|-------------|-------------|-----------------|
| 42 | **83.13%** (69/83) | 81.93% (68/83) | **-1.20 pp** |
| 11 | **84.34%** (70/83) | 79.52% (66/83) | **-4.82 pp** |
| 8 | 79.52% (66/83) | 80.72% (67/83) | +1.20 pp |
| **均值** | **82.33%** | **80.72%** | **-1.61 pp** |
| **σ** | 2.55 pp | 1.20 pp | — |

**对照基准**：
- 14B 零样本 = **83.13%** (69/83)
- 7B 可复现最佳 = **79.52%** (seed=42, 多教师融合)
- 7B DeepSeek 两阶段峰值 = **81.93%** (seed=11)

### 8.3 关键发现

#### 发现一：Stage 2 GT SFT 对 14B 有害

3 个 seed 中 2 个出现 Stage2 回退，平均损失 -1.61 pp。原因：
- 14B 零样本已 83.13%，Stage 2 用 672 条有限数据做 SFT 导致过拟合
- Loss 在 epoch 2 后已降至 <0.001（完全记忆训练集），Val 不再提升

**结论：对 14B 学生应采用单阶段蒸馏，跳过 Stage 2。**

#### 发现二：Stage 1 蒸馏对 14B 几乎无效

| 对比 | Test Acc | vs 零样本 |
|------|----------|----------|
| 14B 零样本 | 83.13% | — |
| Stage 1 s42 | 83.13% | ±0 |
| Stage 1 s11 | 84.34% | +1.20 pp（+1 题） |
| Stage 1 s8 | 79.52% | -3.61 pp（-3 题） |
| **Stage 1 均值** | **82.33%** | **-0.80 pp** |

Stage 1 蒸馏平均比零样本**低 0.80 pp**。seed=11 的 +1 题可能来自随机波动而非蒸馏增益。

**根因：当前软标签是假的。** Module 13 使用 `prepare_soft_labels.py`，对教师的 hard 预测做均匀 label smoothing（ε=0.25），生成千篇一律的 {0.80, 0.05, 0.05, 0.05, 0.05} 分布——不含任何 dark knowledge。672 条全部标为 `clean_teacher`，其中 94 条（14%）教师答案与 GT 不一致，这些样本的蒸馏信号直接有害。

#### 发现三：与 7B 体系对比

| 方法 | 7B | 14B | 提升 |
|------|-----|-----|------|
| 零样本 | 72.29% | 83.13% | +10.84 pp |
| DeepSeek 蒸馏(均值) | ~76% | 82.33% | +6 pp |
| DeepSeek 蒸馏(峰值) | 81.93% (s11) | **84.34%** (s11) | +2.41 pp |
| 可复现基准 | 79.52% | 83.13% (单Stage1) | +3.61 pp |

**14B Stage 1 seed=11 的 84.34% 是全项目最高分**，但这一优势可能仅来自更强的预训练基线，而非蒸馏知识迁移。

### 8.4 提升空间分析

当前结果距离教师水平（DeepSeek 87.95%）还有 **~5.6 pp**（从 Stage1 均值）或 **~3.6 pp**（从峰值）。这个差距能否缩小？

**制约因素**：

1. **测试集仅 83 题**：每 ±1 题 = ±1.20 pp，3 seed σ=2.55 pp。在此方差下，<2 pp 的方法改进不可验证。
2. **教师标签质量差**：当前使用均匀 label smoothing 伪造的"软标签"（0.80/0.05×4），无 dark knowledge。真实 DeepSeek multivote 9-vote 数据虽存在（`teacher_train_multivote.jsonl`），但 87.4% 为 one-hot（教师 9/9 一致），仅 85 条有真实不确定性。
3. **教师-学生差距太小**：DeepSeek 87.95% vs 14B 83.13%，训练集上教师正确率 86%，学生可能在大多数正确样本上已经"会了"，蒸馏只是在学习教师的错误。
4. **672 条数据不够**：对 14B 来说，672 条 MCQ 的信息量极低，训练 1 epoch 即可记忆全部。

### 8.5 优化方案

针对上述制约因素，有以下优化方向：

#### 方案 C1：使用真实多票软标签 + 过滤有害样本

**动机**：将伪造的 label smoothing 替换为 DeepSeek 9-vote 真实分布，并剔除教师与 GT 不一致的 94 条有害样本。

**操作**：
1. 用 `teacher_train_multivote.jsonl` 替代 `teacher_train_soft.jsonl` 重建 head 数据集
2. 仅保留 `teacher_majority == GT` 的样本（~578 条），其余回退为纯 CE loss (`default_distill_mask=0`)
3. 单阶段蒸馏，跳过 Stage2

**预期**：消除 14% 有害样本的负面影响，但 87.4% 的 one-hot 标签意味着增益有限（≤1-2 pp）。

#### 方案 C2：升级学生至 Qwen3-14B

**动机**：Qwen3-14B 使用 36T tokens 预训练（vs Qwen2.5-14B 的 18T），预训练基线更强。Tokenizer 完全兼容，训练脚本零修改可用。

**操作**：
1. 下载 Qwen3-14B（29.5 GB，H100 95GB 充裕）——已在进行
2. 零样本评估建立基线
3. 用方案 C1 的清洁标签做单阶段蒸馏

**预期**：如果零样本 ≥85%，可能接近教师水平。如果零样本 ~83%（与 Qwen2.5-14B 持平），则增益有限。

#### 方案 C3：Qwen3-14B + CoT 蒸馏（Plan B 重试）

**动机**：Plan B 在 7B 上失败的原因之一是"小模型推理能力不足"。Qwen3 具有原生 `<think>` 推理能力，14B 参数量足以内化推理链。

**操作**：
1. 沿用 Module 12 的 657 条 Doubao CoT 数据
2. 在 Qwen3-14B 上做 CoT SFT 训练
3. 评估时让模型先 `<think>` 推理再给答案

**预期**：高风险高收益。若成功可同时提升准确率和可解释性（毕业论文加分项）。

#### 方案 C4：扩大训练数据

**动机**：672 条对 14B 来说信息量不足。可从 CMExam 的其他科室抽取更多医学 MCQ 做通用医学 SFT 预训练，再在牙科子集上微调。

**操作**：需要额外数据采集，超出当前实验范围。

**暂不推荐**。

### 8.6 推荐执行顺序

```
优先级 1: 方案 C2 — Qwen3-14B 零样本基线 (0成本，等下载完即可评估)
    ↓
优先级 2: 方案 C1 — 清洁标签单阶段蒸馏 (对 Qwen3-14B 或 Qwen2.5-14B)
    ↓  
优先级 3: 方案 C3 — Qwen3-14B + CoT 蒸馏 (Plan B 重试)
```
