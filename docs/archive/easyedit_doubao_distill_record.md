## 2. 教师数据准备

### 2.1 教师标签获取

通过火山引擎 ARK API 对训练集题目进行推理，获取教师标签及软概率分布。

- **API 端点**：`https://ark.cn-beijing.volces.com/api/v3/chat/completions`
- **认证方式**：API Key（ARK 平台 Bearer Token）
- **教师标签匹配数**：604/672（89.88% 覆盖率，68 题未成功匹配）
- **教师-真值不一致率**：0.33%（仅 2/604 样本教师答案与标准答案不同）

### 2.2 软标签生成：多投票（MV5）方法

豆包采用多投票+软标签的方式生成教师概率分布，与 DeepSeek 的 label smoothing 方法不同：

1. **多投票推理**：对每道题目调用豆包 API 5 次（MV5 = Multi-Vote × 5）
2. **概率聚合**：统计 5 次推理中各选项被选择的频率作为软概率分布
3. **答案确定**：选择最高频率的选项作为教师答案

**示例**：5次推理结果为 [A, A, A, B, A] → {A: 0.80, B: 0.20, C: 0.00, D: 0.00, E: 0.00}

软标签平均熵：0.7777，峰值平均值：0.80。

### 2.3 教师数据质量对比

| 指标 | 豆包 | DeepSeek-V3 |
|------|------|-------------|
| 教师测试准确率 | **98.80%** | 87.95% |
| 训练集覆盖率 | 89.88% (604/672) | **100% (672/672)** |
| 教师-GT 不一致率 | **0.33% (2/604)** | 14.14% (95/672) |
| 软标签方法 | 多投票聚合 (MV5) | Label Smoothing (ε=0.25) |
| 软标签信息量 | 低（几乎等于GT） | 高（含14%冲突信息） |

> **关键差异**：豆包教师的不一致率仅 0.33%，意味着其软标签本质上接近于真值硬标签的平滑版本。蒸馏的"新信息"极少，导致蒸馏增益有限但稳定性较好。相比之下，DeepSeek 有 14.14% 的标签冲突，提供更多"新信息"但也引入更多噪声。

## 3. 蒸馏方法

### 3.1 损失函数

采用加权组合损失（与 DeepSeek 实验一致）：

$$\mathcal{L} = \alpha \cdot \text{KL}\big(P_{\text{student}}, P_{\text{teacher}}\big) + (1 - \alpha) \cdot \text{CE}\big(P_{\text{student}}, y_{\text{GT}}\big)$$

其中：
- $\alpha$ 为蒸馏权重（0～1）
- KL 散度在选项 token（A-E）位置上计算（Choice-Head 方法）
- CE 为标准交叉熵损失

### 3.2 训练方案演进

豆包蒸馏共经历 9 轮迭代，方案不断优化：

| 阶段 | 轮次 | 方法 | 说明 |
|------|------|------|------|
| 探索期 | R1 | 两阶段 KL 蒸馏 | 首次尝试，全部低于基线 |
| 探索期 | R2 | 教师标签纯 SFT | 用教师答案直接训练，峰值 80.72% |
| 调参期 | R3 | seed+lr 邻域搜索 | seed=42 可复现 80.72% |
| 调参期 | R4 | 进一步窄搜 | 最佳 79.52% |
| 软标签期 | R5 | 全序列 KL 蒸馏 | 使用 MV5 软标签，效果不佳 |
| 混合期 | R6/R6b | 选择性混合蒸馏 | 混合 KL + CE，未改善 |
| **突破期** | **R7** | **Choice-Head → SFT** | **首次系统性超越基线** |
| 验证期 | R8 | 调参变体 | α=0.25 前轮设置劣化 |
| **终结** | **R9** | **5-seed 稳定性验证** | **最终验证 R7 真实稳定性** |

### 3.3 最终方案：Choice-Head → SFT 两阶段

| 阶段 | 目标 | Epochs | α 值 | 说明 |
|------|------|--------|------|------|
| **Stage 1：Choice-Head 蒸馏** | 校准选项级分类头 | 1 | 0.35 | 仅在 A-E token 位置计算 KL 损失 |
| **Stage 2：GT 生成式 SFT** | 恢复全序列生成能力 | 2 | 0.0 | 从 Stage 1 的 LoRA 权重继续训练 |

### 3.4 LoRA 微调参数（最终方案）

| 参数 | 值 |
|------|------|
| LoRA Rank | 16 |
| LoRA Alpha | 32 |
| 有效缩放因子 | α/r = 32/16 = 2.0 |
| 目标模块 | q_proj, v_proj |
| Batch Size | 2 |
| 梯度累积步数 | 4 |
| 有效批大小 | 2 × 4 = 8 |
| 数据类型 | bfloat16 |

### 4.2 各阶段详细结果

#### Round 1：两阶段 KL 蒸馏（全部失败）

首次尝试直接用豆包软标签进行蒸馏。

| 配置 | α | T | lr | 准确率 |
|------|---|---|-----|--------|
| a008_lr15 | 0.08 | 1.5 | 1.5e-4 | 73.49% |
| a012_lr15 | 0.12 | 1.5 | 1.5e-4 | 73.49% |
| a016_lr15 | 0.16 | 1.5 | 1.5e-4 | 73.49% |
| a004_lr10 | 0.04 | 1.5 | 1.0e-4 | 72.29% |
| a004_lr15 | 0.04 | 1.5 | 1.5e-4 | 72.29% |
| a012_lr10 | 0.12 | 1.5 | 1.0e-4 | 72.29% |
| a008_lr10 | 0.08 | 1.5 | 1.0e-4 | 71.08% |
| a016_lr10 | 0.16 | 1.5 | 1.0e-4 | 69.88% |

**结论**：8 组配置全部低于基线（77.11%），最差仅 69.88%。豆包软标签的直接 KL 蒸馏无效。

#### Round 2：教师标签纯 SFT（首次突破）

转变思路：不做 KL 蒸馏，直接用豆包的预测标签替代 GT 做 SFT。

| 配置 | lr | Epochs | 准确率 |
|------|-----|--------|--------|
| **sft_lr15_ep3** | **1.5e-4** | **3** | **80.72%** |
| sft_lr10_ep3 | 1.0e-4 | 3 | 78.31% |
| sft_lr08_ep3 | 0.8e-4 | 3 | 73.49% |
| sft_lr10_ep4 | 1.0e-4 | 4 | 72.29% |

**结论**：纯教师标签SFT达到 80.72%（+3.61pp），但此结果未控制随机种子。

#### Round 3：多 seed 复现验证

对 R2 最佳配置进行多组 seed 验证。

| 配置 | lr | Seed | 准确率 |
|------|-----|------|--------|
| **sft_lr15_ep3_s42** | **1.5e-4** | **42** | **80.72%** |
| sft_lr13_ep3_s7 | 1.3e-4 | 7 | 78.31% |
| sft_lr12_ep4_s33 | 1.2e-4 | 33 | 77.11% |
| sft_lr15_ep3_s11 | 1.5e-4 | 11 | 74.70% |
| sft_lr15_ep3_r32_s42 | 1.5e-4 | 42 | 74.70% |
| sft_lr17_ep3_s21 | 1.7e-4 | 21 | 73.49% |
| sft_lr15_ep2_s42 | 1.5e-4 | 42 | 73.49% |
| sft_lr10_ep3_r32_s55 | 1.0e-4 | 55 | 71.08% |

**结论**：seed=42 可复现 80.72%，但其他 seed/配置大幅下降。80.72% 可能是局部最优。

#### Round 4：窄搜验证

在 R3 最佳配置周围窄范围搜索。

| 配置 | lr | Seed | Rank | 准确率 |

#### Round 5：MV5 全序列软标签蒸馏

使用 MV5 多投票真实软标签进行全序列 KL 蒸馏。

| 配置 | α | lr | Rank | 准确率 |
|------|---|-----|------|--------|

**结论**：MV5 真实软标签的全序列蒸馏效果不佳。α 越大，效果越差。

#### Round 6/6b：选择性混合蒸馏

混合蒸馏策略，对不同样本使用不同的 KL/CE 权重。

## 6. 综合分析

### 6.1 方法迭代总结

| 方法类别 | 轮次 | 最佳准确率 | 均值 | 结论 |
|----------|------|-----------|------|------|
| 直接 KL 蒸馏 | R1 | 73.49% | 72.17% | 完全失败 |
| 教师标签纯 SFT | R2, R3, R4 | 80.72% | ~76% | 峰值高但不稳定 |
| MV5 全序列软标签蒸馏 | R5 | 77.11% | 75.00% | 无增益 |
| 选择性混合蒸馏 | R6, R6b | 74.70% | 73.90% | 负效果 |
| **Choice-Head → SFT** | **R7, R8, R9** | **79.52%** | **77.35%** | **稳定超基线** |

### 6.2 关键发现

1. **教师信息量不足是核心瓶颈**：豆包教师的 GT 不一致率仅 0.33%，软标签几乎等价于真值。蒸馏无法从中提取有意义的新信息。

2. **Choice-Head 方法的贡献机制**：在选项 token（A-E）位置的概率校准比全序列 KL 更有效。这可能是因为多选题任务的关键判别信号集中在这 5 个token 上，而非整个生成序列。

3. **方差为主要挑战**：R9 的 5-seed 结果范围为 74.70%～79.52%（跨度 4.82pp），远大于方法间的差异。这表明在当前数据规模（604 样本）下，随机种子对结果的影响权重很高。

4. **R7 的"高稳定性"是幻觉**：3 seeds 的 mean=79.12%（std=0.57）看似极度稳定，但扩展到 5 seeds 后 mean 降至 77.35%（std=2.21）。**种子选择偏差会严重高估方法有效性**。

5. **教师标签纯 SFT 的80.72%不可靠**：仅在特定 seed+lr 组合下出现，无法系统复现。

### 6.3 与 DeepSeek 教师对比

| 指标 | 豆包 (R9) | DeepSeek (Opus R1) |
|------|-----------|-------------------|
| **最佳准确率** | 79.52% | **81.93%** |
| **均值** | **77.35%** | 76.87% |
| **标准差** | **2.21** | 3.09 |
| **超基线率** | **60% (3/5)** | 20% (1/5) |
| **教师-GT 不一致率** | 0.33% | 14.14% |
| **特点** | 保守稳定，上限有限 | 冒险激进，峰值更高 |

**核心权衡**：豆包提供更稳定的结果（60% 超基线率），但峰值受限（79.52%）；DeepSeek 提供更高的上限（81.93%），但可靠性差（仅 20% 超基线）。

### 6.4 论文推荐表述

> 豆包教师蒸馏采用 Choice-Head 两阶段方案，在 5 个随机种子的验证下取得平均准确率 77.35%（std=2.21），最佳 79.52%，超基线率 60%。相较 Qwen2.5-7B-Instruct 独立测试基线 77.11%，稳定可复现的最优成绩为 79.52%（+2.41pp，相对 +3.12%）。与 DeepSeek 教师相比，豆包蒸馏的均值更高（77.35% vs 76.87%）且方差更低（2.21 vs 3.09），但峰值低（79.52% vs 81.93%），呈现出"稳定但上限有限"的特征。分析表明，豆包教师极低的标签不一致率（0.33%）限制了蒸馏能够提供的知识增量。

## 7. 9 轮迭代的经验教训

| 教训 | 对应证据 |
|------|----------|
| 直接 KL 蒸馏在教师标签接近 GT 时无效 | R1 全部低于基线 |
| 教师标签替代 GT 做 SFT 可产生高峰值 | R2 达到 80.72% |
| 高峰值可能来自 seed 偶然性而非方法优越 | R3/R4 无法稳定复现 80.72% |
| 真实软标签（MV5）比 label smoothing 并不更好 | R5 效果不如 R2 纯 SFT |
| 全序列 KL 不如 Choice-Head（选项级）蒸馏 | R6 vs R7 |
| α₁ = 0.35 是 Choice-Head 蒸馏的关键参数 | R8 改为 0.25 后严重劣化 |
| 少 seed 验证会高估方法稳定性 | R7(3-seed)→R9(5-seed) 均值降 1.77pp |
| 数据规模 (604 样本) 下随机性主导结果 | R9 标准差 2.21 > 方法间差异 |
