## ✨ 核心改动详解

### 改动 1: 单目标训练（★最关键）
**问题**:
```python
# ❌ 原来的双目标蒸馏
target = f"{teacher_ans}|{new_answer}"  # 如 "A|E"，当两个答案不同时产生梯度冲突
```

**解决**:
```python
# ✅ 改为单目标训练
target = new_answer  # 仅 "A"，单一清晰目标
```

**效果**: 
- 消除梯度冲突 → **更快收敛**
- 与评估指标一致 → **准确率虚高消除**
- **预期提升: +5-8%**

### 改动 2: 保持选项字母固定
**问题**:

```python
# ❌ 原来打乱选项，导致 A/B/C/D/E 每次顺序都不同
shuffled_options = options.copy()
random.shuffle(shuffled_options)  # 选项字母被打乱
```

**解决**:
```python
# ✅ 保持 A/B/C/D/E 固定
option_text = "\n".join([f"{opt[0]}. {opt[1]}" for opt in options])
new_answer = original_answer
```

**效果**:

## 🔍 技术细节

### 为什么这些改动有效？

**1. 消除梯度冲突**
- 原来: 模型被让一起输出 "A" 和 "E" → 梯度冲突 → 收敛困难
- 改后: 模型只输出确定的一个答案 → 清晰目标 → 快速收敛

**2. 与评估指标对齐** 
- autoTest7.py: 期望单字母 "A"
- 原训练: 模型被教导输出 "A|E"
- 改后: 训练和评估目标一致 ✓

**3. 模型容量扩大**
- LoRA rank 16→32: 参数增加 2 倍，学习能力增强
- steps 200→500: 充分利用数据，防止欠拟合

**4. 学习率提升**
- lr 5e-5→1e-4: 加速收敛，在给定步数内学到更多
- 但保留正则化防止过拟合
