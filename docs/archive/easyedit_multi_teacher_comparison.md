# 多教师模型知识蒸馏对比分析

## 1. 实验背景

### 1.1 研究目标

以 **Qwen2.5-7B-Instruct** 作为学生模型，探索不同教师模型对牙科选择题任务知识蒸馏效果的影响。通过系统对比 6 种教师模型（含同系列不同规模模型和外部闭源 API 模型），分析教师模型的能力、标签质量与蒸馏增益之间的关系，寻找最优蒸馏方案。

### 1.2 实验设置

| 项目 | 内容 |
|------|------|
| **学生模型** | Qwen2.5-7B-Instruct（7B 参数，bf16 精度） |
| **任务** | 牙科单项选择题（A/B/C/D/E 五选一） |
| **训练集** | CMExam 牙科考试题 672 道（固定训练集，无泄漏） |
| **验证集** | 74 道（与训练集/测试集不重叠） |
| **测试集** | 83 道（固定金标准集） |
| **硬件** | NVIDIA H100 80GB GPU |
| **精度** | bf16 混合精度 |
| **微调方法** | LoRA（rank=16, alpha=32, target: q/k/v/o_proj） |
| **评估方式** | 确定性推理（do_sample=False），仅评价首字母 |

### 1.3 基线

**学生模型基线（纯 GT SFT）**：在训练集上使用标准答案进行监督微调后，测试集准确率为 **77.11%**（64/83）。此为所有蒸馏实验的对照基准。

| 教师模型 | 类型 | 参数规模 | 测试集准确率 | 正确题数/总题数 |
|----------|------|----------|-------------|----------------|

## 3. 各教师模型蒸馏结果详情

### 3.1 Qwen2.5-32B-Instruct → 7B（同系列白盒蒸馏）

| 项目 | 内容 |
|------|------|
| **教师测试准确率** | 83.13% |
| **蒸馏方式** | 白盒蒸馏（Logit-level KL 散度） |
| **最佳测试准确率** | **80.72%** |
| **训练脚本** | `train_dental_lora32.py` |

#### 最佳配置参数

| 参数 | 值 |
|------|------|
| 学习率 | 1e-4 |
| 训练轮次 | 4 |
| 批大小 | 1 |
| 梯度累积 | 8 |
| LoRA rank | 16 |
| LoRA alpha | 32 |
| 蒸馏温度 T | 2.0 |
| KL 权重 α | 0.1（warmup 1 epoch） |
| 种子 | 2 |
| 困难样本上采样 | 2× |

#### 方案说明

采用经典的 **Logit-level 白盒蒸馏**：教师（32B）和学生（7B）同时加载到 GPU，前向传播后计算 softmax 温度缩放后的 KL 散度损失，与交叉熵损失加权求和。α 使用 warmup 策略（第一轮纯 CE，之后引入 KL）。同时使用困难样本挖掘（教师答对但学生答错的题目）进行 2× 上采样。

#### 关键发现

- 白盒蒸馏可利用完整的教师 logit 概率分布，信息量最大
- 80.72% 结果依赖特定种子（seed=2），其他种子可能回落
- 教师-学生能力差距适中（83.13% vs 77.11%），蒸馏空间约 6 个百分点

### 3.2 DeepSeek-V3 → 7B（外部 API 黑盒蒸馏）

| 项目 | 内容 |
|------|------|
| **教师测试准确率** | 87.95%（73/83） |
| **教师训练集覆盖率** | 100%（672/672） |
| **教师-GT 不一致率** | 14.14%（95/672 样本教师答案与标准答案不同） |
| **蒸馏方式** | 黑盒蒸馏（Label Smoothing 构造软标签 + Choice-Head KL） |
| **最佳测试准确率** | **81.93%**（历史最高） |
| **稳定可复现最佳** | **79.52%** |
| **训练脚本** | `train_dental_choice_head_distill_doubao.py`（Stage 1）+ `train_dental_lora7_deepseek.py`（Stage 2） |

#### 最佳配置参数（两阶段 Choice-Head → SFT）

| 参数 | Stage 1（Choice-Head KL） | Stage 2（GT SFT） |
|------|--------------------------|-------------------|
| 学习率 | 1.2e-4 | 1.2e-4 |
| 训练轮次 | 1 | 2 |
| KL 权重 α | 0.35 | 0.0（纯 SFT） |
| LoRA rank | 16 | 16（从 Stage 1 续训） |
| LoRA alpha | 32 | 32 |
| 批大小 | 2 | 2 |
| 梯度累积 | 4 | 4 |
| 种子 | 11（峰值）/ 7,99（稳定） | 同上 |

#### 软标签构造

由于 DeepSeek-V3 为 API 调用（黑盒），无法获取完整 logit 分布，采用以下方式构造软标签：
1. 获取教师硬标签（672 题全覆盖）
2. 使用 **Label Smoothing（ε=0.25）** 生成软概率分布
3. 通过 `prepare_soft_labels.py` 转换为 `TeacherDist` 字段

#### 两阶段训练流程

1. **Stage 1（Choice-Head 蒸馏）**：仅在 A/B/C/D/E 五个选项 token 上计算 KL 散度，学习教师的选项概率分布
2. **Stage 2（GT SFT 续训）**：以 Stage 1 的 LoRA 权重为起点，使用标准答案进行纯 SFT 训练，巩固正确知识

#### 多种子验证结果

| 种子 | 测试准确率 | vs 基线 |
|------|----------|---------|
| 11 | **81.93%** | +4.82 |
| 7 | 77.11% | ±0.00 |
| 99 | 77.11% | ±0.00 |
| 42 | 72.29% | -4.82 |
| 55 | 75.90% | -1.21 |
| **均值** | **76.87%** | -0.24 |
| **标准差** | **3.09%** | — |

#### 关键发现

- 峰值 81.93% 仅在 seed=11 时出现（20% 种子成功率）
- 14.14% 的教师-GT 不一致率提供了最丰富的蒸馏信号（多样性知识）
- 平均准确率反而低于基线，体现了"高上限, 低稳定性"的特点
- **稳定可复现的最佳结果为 79.52%**（低 α 单阶段方案，α=0.08, T=1.5, lr=1.0e-4）

### 3.3 豆包（Doubao）→ 7B（外部 API 黑盒蒸馏）

| 项目 | 内容 |
|------|------|
| **教师测试准确率** | 98.80%（82/83） |
| **教师训练集覆盖率** | 89.88%（604/672） |
| **教师-GT 不一致率** | 0.33%（仅 2/604 样本不一致） |
| **蒸馏方式** | 黑盒蒸馏（MV5 多投票软标签 + Choice-Head KL） |
| **最佳峰值准确率** | **80.72%**（seed=42, 纯教师 SFT） |
| **稳定可复现最佳** | **79.52%**（Choice-Head 两阶段） |
| **训练脚本** | 同 DeepSeek 两阶段流程 |

#### 最佳稳定配置参数（Two-Stage Choice-Head → SFT）

| 参数 | Stage 1 | Stage 2 |
|------|---------|---------|
| 学习率 | 1.2e-4 | 1.2e-4 |
| 训练轮次 | 1 | 2 |
| KL 权重 α | 0.35 | 0.0 |
| LoRA rank | 16 | 16 |

#### 软标签构造（MV5 多投票）

与 DeepSeek 不同，豆包采用 **多投票聚合** 构造软标签：
1. 对每道题调用豆包 API **5 次**（MV5 = Multi-Vote × 5）
2. 统计各选项被选中的频率作为软概率分布
3. 软标签平均熵：0.7777，峰值平均值：0.80

#### 多种子验证结果（Choice-Head 两阶段）

| 种子 | 测试准确率 | vs 基线 |
|------|----------|---------|
| 42 | **79.52%** | +2.41 |
| 55 | 79.52% | +2.41 |
| 11 | 78.31% | +1.20 |
| 7 | 74.70% | -2.41 |
| 99 | 74.70% | -2.41 |
| **均值** | **77.35%** | +0.24 |
| **标准差** | **2.21%** | — |

#### 关键发现

- 60% 种子成功率（3/5 ≥ 78.31%），稳定性优于 DeepSeek
- 均值 77.35% 高于 DeepSeek 的 76.87%，方差更低（2.21 vs 3.09）
- 但峰值有限（79.52% vs 81.93%），呈现 **"稳定但上限有限"** 的特征
- 极低的教师-GT 不一致率（0.33%）从根本上限制了蒸馏增益
- 纯教师 SFT 80.72% 仅在 seed=42 + lr=1.5e-4 特定组合下出现，不可系统复现

## 5. 分析与讨论

### 5.1 为什么准确率上限维持在 ~82%，难以突破？

经过 6 种教师模型、超过 50 组超参数配置、多种蒸馏策略的系统实验，测试集准确率的天花板始终在 81.93% 左右。这一现象可以从以下几个维度进行分析：

#### （1）训练数据量不足——672 样本的根本瓶颈

当前训练集仅有 **672 道**牙科选择题，对于覆盖口腔医学全部知识点（涵盖口腔内科、口腔外科、修复、正畸、儿牙、预防等十余个方向的数百个知识点）而言，数据量严重不足。

- 测试集 83 题中每道题对应的知识点可能在训练集中仅出现 1-2 次甚至完全缺失
- 部分错题涉及"阈值/范围"类知识（如温度测验正常值区间、X 线特征性表现），这类精确知识点在少量训练样本中很难被模型稳定学习
- **数据增强实验的佐证**：将训练数据从 672 扩充至 1033（加入 HuaTuo 改写 + 定向知识点补充），纯 SFT 也可达到 81.93%，说明数据量本身是关键限制

#### （2）测试集规模小——统计噪声放大

测试集仅 83 道题，**每道题的权重为 1.20 个百分点**。这意味着：

- 多答对 1 题 = +1.20%，少答对 1 题 = -1.20%
- 81.93%（68 题正确）与 77.11%（64 题正确）之间仅差 **4 道题**
- 不同随机种子导致的训练路径差异就足以影响 2-4 道边界题的正确性
- 这解释了为什么种子敏感性如此显著——本质是测试集过小导致的统计波动

#### （3）教师知识增量有限——"知道的都差不多"

| 教师模型 | 教师-GT 不一致率 | 蒸馏增益 |
|----------|----------------|---------|
| DeepSeek-V3 | 14.14% | +4.82（峰值） |
| 豆包 | 0.33% | +2.41 |
| Kimi | 0% | ±0.00 |

存在清晰的正相关：**教师与 GT 越不一致（即教师提供了更多"不同视角"的知识），蒸馏增益越大**。

- DeepSeek 有 14% 的不一致，提供了最丰富的蒸馏信号，但也引入了教师噪声
- 豆包几乎完全与 GT 一致，蒸馏软标签约等于 GT 的平滑版，增益有限
- Kimi 训练集上 100% 与 GT 一致，但自身测试能力很弱（61.45%），说明它只是"碰巧"在训练集上答对，没有真正的知识可传递

**结论**：对于这 83 道测试题，教师能额外"教会"学生的知识空间本身就很窄。

#### （4）LoRA 微调的表达能力上限

所有实验均使用 LoRA（rank=16, alpha=32）进行参数高效微调，仅更新约 **0.3%** 的模型参数。LoRA 的优势是训练高效，但也限制了模型的学习容量：

- rank=16 的低秩分解可能无法捕获细粒度的专业医学知识修正
- 增大 rank（如 32）的额外实验中并未观察到显著提升，说明瓶颈可能不完全在 LoRA 容量

#### （5）任务本身的"可学习性"上限

test set 中约 14-16 道题（17%~19%）属于以下类型，当前方案固有难以解决：

- **精确数值/阈值题**：如"牙髓活力测试的正常温度范围"，需要精确记忆而非推理
- **易混淆邻近概念**：如"上颌窦与上颌第一磨牙的毗邻关系"vs "上颌第二磨牙"
- **否定/排除题型**：如"以下哪项不正确"，需要逻辑取反，7B 模型的否定推理能力受限
- **低频知识点**：训练集中仅出现 0-1 次的知识点，无论教师多强也无法从训练数据中学到

### 5.2 教师模型能力与蒸馏效果的非线性关系

一个反直觉的发现是：**教师模型并非越强越好**。

| 教师 | 教师准确率 | 学生最佳准确率 | 蒸馏增益 |
|------|----------|--------------|---------|
| Kimi | 61.45% | 77.11% | ±0.00 || **Qwen-14B** | **77.11%** | **75.90%** | **-1.21** || Qwen-32B | 83.13% | 80.72% | +3.61 |
| DeepSeek | 87.95% | 81.93% | +4.82 |
| 豆包 | **98.80%** | 80.72% | +3.61 |

- 教师太弱（Kimi 61.45%）→ 无法提供有用信号
- 教师最强（豆包 98.80%）→ 与 GT 几乎完全一致，软标签退化为"花式 GT"，增益有限
- 教师适度偏强且有一定"个性"（DeepSeek 87.95%, 14% 不一致）→ 蒸馏增益最大

Qwen2.5-14B 的加入进一步验证了这一规律：14B 教师准确率与 7B 相同（77.11%），但训练集不一致率高达 20.24%——然而这些不一致来自教师的错误而非"更优视角"，反而引入了有害噪声，导致蒸馏后准确率下降（75.90%）。

这表明，**有效蒸馏需要教师同时具备"正确性"和"多样性"**——教师在大部分题目上正确（确保知识可靠），同时在部分题目上给出与 GT 不同但合理的答案分布（提供额外信息）。仅有"多样性"（如14B的高不一致率）但缺乏"正确性"优势时，蒸馏反而有害。

### 5.3 两阶段方案为何有效？

两阶段 Choice-Head → SFT 方案在 DeepSeek 和豆包教师下均为最佳稳定方案，其优势在于：

1. **Stage 1 聚焦选项分布**：仅在 A/B/C/D/E 五个 token 上做 KL 对齐，避免全词表 softmax 的噪声干扰，信号更集中
2. **Stage 2 回归正确标签**：在教师知识初始化的基础上用 GT 进行纠偏，防止教师错误答案的累积
3. **LoRA 权重续训**：Stage 2 从 Stage 1 的 LoRA 权重开始训练，保留了教师传递的知识先验

### 5.4 稳定性与峰值的权衡

| 口径 | 最佳教师 | 准确率 | 特点 |
|------|---------|--------|------|
| 峰值口径 | DeepSeek-V3 | 81.93% | 仅 20% 种子可达 |
| 稳定口径 | DeepSeek-V3 / 豆包 | 79.52% | 多种子可复现 |

所有超过基线的结果都表现出显著的**种子敏感性**。这并非实验缺陷，而是反映了：
- 数据量小 → 损失面多个局部最优 → 不同种子落入不同最优区域
- 测试集小 → 微小参数差异就能翻转 1-2 道边界题的正确性

## 6. 结论

### 6.1 核心结论

1. **最佳蒸馏方案**：DeepSeek-V3 → 7B 两阶段 Choice-Head→SFT，峰值 81.93%（+4.82pp），为全部实验最高值
2. **最稳定方案**：DeepSeek-V3 / 豆包 → 7B，稳定达到 79.52%（+2.41pp）
3. **教师选择**：教师模型需兼具正确性与多样性，"适度偏强"优于"极致正确"
4. **弱教师无效**：Kimi（61.45%）证明低于学生能力的教师无法通过蒸馏提供增益
5. **同等教师有害**：Qwen2.5-14B（77.11%）与学生能力相同，蒸馏后反而下降至 75.90%，高不一致率（20.24%）引入的是噪声而非知识
6. **瓶颈定位**：准确率上限（~82%）受限于训练数据量（672 样本）、测试集统计波动（83 题）和任务固有难度（精确知识点覆盖不足）

### 6.2 提升方向建议

| 方向 | 预期收益 | 可行性 |
|------|---------|--------|
| 扩充训练数据（目标 2000+） | 高 | 需收集更多牙科考题 |
| 扩大测试集（目标 300+） | 中（降低统计波动） | 需更多标注资源 |
| 全参数微调替代 LoRA | 中 | 需更多 GPU 显存 |
| 多教师集成蒸馏 | 中 | 结合 DeepSeek + 豆包的互补优势 |
| 更强基座模型（14B 学生） | 高 | 需更多计算资源 |
