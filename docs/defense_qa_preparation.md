> 本文档整理项目涉及的所有核心技术概念，并模拟答辩老师可能提出的问题，附带建议回答。

---

## 第一部分：核心技术概念手册

---

### 1. 知识蒸馏（Knowledge Distillation, KD）

**定义**：由 Hinton 等人（2015）提出的模型压缩技术。核心思想是让一个小模型（学生）模仿大模型（教师）的输出分布，从而将教师的"暗知识"（dark knowledge）迁移到学生中。

**暗知识（Dark Knowledge）**：教师模型在非正确选项上的概率分布包含了类间相似性信息。例如教师对一道题输出 `[A:0.85, B:0.10, C:0.03, D:0.01, E:0.01]`，其中 B 拿到 0.10 说明 B 和正确答案 A 有一定相似性，这比 one-hot 标签 `[1,0,0,0,0]` 提供了更丰富的监督信号。

**标准 KD 损失**：

$$\mathcal{L}_{\text{KD}} = \alpha \cdot T^2 \cdot D_{\text{KL}}(\sigma(\mathbf{z}_T/T) \| \sigma(\mathbf{z}_S/T)) + (1-\alpha) \cdot \mathcal{L}_{\text{CE}}(y^*, \mathbf{z}_S)$$

- $\mathbf{z}_T, \mathbf{z}_S$：教师/学生的 logits
- $T$：温度参数，控制分布平滑度
- $\alpha$：KL 损失权重
- $\sigma$：softmax 函数

---

### 2. 温度参数（Temperature, T）

**作用**：对 logits 除以温度 $T$ 后再做 softmax，使概率分布更"平滑"。

$$p_i = \frac{\exp(z_i / T)}{\sum_j \exp(z_j / T)}$$

- $T = 1$：标准 softmax
- $T > 1$：分布变平滑，暗知识更多地暴露出来（非正确选项的概率被放大）
- $T \to \infty$：趋向均匀分布
- $T \to 0$：趋向 one-hot（argmax）

**本项目使用**：白盒蒸馏中 $T=2.0$；Choice-Head 蒸馏中未额外加温度（$T=1$），因为软标签本身已是概率分布。

---

### 3. KL 散度（Kullback-Leibler Divergence）

**定义**：衡量两个概率分布之间差异的非对称度量：

$$D_{\text{KL}}(P \| Q) = \sum_i P(i) \log \frac{P(i)}{Q(i)}$$

- $D_{\text{KL}}(P \| Q) \geq 0$，等号当且仅当 $P = Q$
- **非对称**：$D_{\text{KL}}(P \| Q) \neq D_{\text{KL}}(Q \| P)$
- 在蒸馏中，$P$ = 教师分布（固定），$Q$ = 学生分布（待优化）

**为什么用 KL 而不是 MSE**：KL 散度直接作用于概率分布，能更好地衡量分布形状的匹配度；MSE 作用于标量概率值，忽略了分布的全局结构。

---

### 4. 交叉熵损失（Cross-Entropy Loss, CE）

**定义**：分类任务的标准损失函数：

$$\mathcal{L}_{\text{CE}} = -\sum_i y_i \log p_i$$

当标签是 one-hot 时简化为 $\mathcal{L}_{\text{CE}} = -\log p_{y^*}$，即正确类别的负对数概率。

**与 KL 散度的关系**：$D_{\text{KL}}(P \| Q) = H(P, Q) - H(P)$，其中 $H(P,Q)$ 是交叉熵，$H(P)$ 是 $P$ 的熵。当 $P$ 固定时，最小化 KL 等价于最小化交叉熵。

---

### 5. 白盒蒸馏 vs 黑盒蒸馏

| 维度 | 白盒蒸馏 | 黑盒蒸馏 |
|------|---------|---------|
| 教师访问 | 完整模型权重，可前向传播 | 仅 API 输出（文本/概率） |
| 蒸馏信号 | 全词表 logit 分布（151K 维） | 选项概率/文本答案 |
| 显存需求 | 极高（需同时加载Teacher+Student） | 低（教师离线生成标签） |
| 适用场景 | 开源本地模型 | 商业 API 模型 |
| 信息量 | 最大（vocab级别） | 较小但更聚焦 |

**本项目**：
- 白盒：Module 01，32B+7B 同时加载，~90GB 显存
- 黑盒：Module 02-08，教师通过 API 生成 5 选项概率分布

---

### 6. Choice-Head（选项级分类头蒸馏）

**本项目核心方法**。不同于标准白盒蒸馏对全词表（vocab=151,936）做 KL，Choice-Head 只提取学生模型在 A/B/C/D/E 五个 token 上的 logit/概率，与教师提供的五选项概率做 KL。

**为什么叫"Choice-Head"**：
- "Choice" = 选项（A/B/C/D/E）
- "Head" = 分类头（常规分类网络最后一层叫 head）
- 概念上等价于在 LLM 的输出层上叠加了一个 5-way 分类头

**实现方式**：
```python
# 提取学生在 A/B/C/D/E token_id 处的 logit
choice_logits = student_logits[:, [token_A, token_B, token_C, token_D, token_E]]
student_probs = softmax(choice_logits)
loss_kl = KL(teacher_probs, student_probs)
```

**优势**：
1. 与任务完美对齐（选择题只关心 5 个选项）
2. 消除了全词表中 151,931 个无关 token 的噪声梯度
3. 可使用黑盒 API 教师（只需要 5 个概率值）

---

### 7. 两阶段训练策略（Two-Stage Training）

**Stage 1 — Choice-Head KL 蒸馏（1 epoch）**：
- 损失 = $\alpha \cdot D_{\text{KL}}(p_T \| p_S) + (1-\alpha) \cdot \mathcal{L}_{\text{CE}}$
- 目标：从教师分布中学习"暗知识"
- 仅 1 epoch 防止过度拟合教师错误

**Stage 2 — GT SFT 精校（2-5 epochs）**：
- 损失 = $\mathcal{L}_{\text{CE}}(y^*, p_S)$ （纯标准答案监督）
- 目标：继承 Stage 1 权重后，用 GT 标签校准，纠正教师引入的错误
- 继承 Stage 1 的 LoRA adapter，不重新初始化

**设计思想**：先从教师学"怎么思考"（概率分布形状），再用标准答案纠正"哪里想错了"。

---

### 8. LoRA（Low-Rank Adaptation）

**定义**：Hu 等人（2022）提出的参数高效微调方法。冻结预训练模型权重 $W_0$，仅训练低秩分解矩阵 $\Delta W = BA$：

$$W = W_0 + \frac{\alpha_{\text{lora}}}{r} \cdot BA$$

- $W_0 \in \mathbb{R}^{d \times d}$：冻结的原始权重
- $B \in \mathbb{R}^{d \times r}$，$A \in \mathbb{R}^{r \times d}$：可训练的低秩矩阵
- $r$：秩（rank），远小于 $d$
- $\alpha_{\text{lora}}$：缩放因子

**参数效率**：

| 模型 | 全参数微调 | LoRA (r=16) | 压缩比 |
|------|-----------|-------------|--------|
| 7B | 7.6B | ~26M | 0.34% |
| 14B | 14.7B | ~52M | 0.35% |

**本项目统一配置**：`rank=16, lora_alpha=32, target_modules=[q_proj, v_proj]`

- `rank=16`：每层有 16 维的可学习子空间
- `lora_alpha=32`：缩放倍数 = $\alpha/r = 32/16 = 2$
- `target_modules`：仅在 Attention 的 Q 和 V 投影矩阵上加 LoRA

---

### 9. 软标签 vs 硬标签

| 类型 | 形式 | 信息量 | 示例 |
|------|------|--------|------|
| 硬标签（Hard Label） | one-hot | 1 bit | `[0, 0, 1, 0, 0]` → 答案是 C |
| 软标签（Soft Label） | 概率分布 | ~2.3 bits | `[0.05, 0.10, 0.70, 0.10, 0.05]` |
| GT（Ground Truth） | 标准答案 | 1 bit | 答案 = C |

**软标签的价值**：包含选项间的相对关系。如 `B:0.10` 说明 B 是第二可能答案，这种"排序信息"对学生学习有帮助。

**软标签退化**：当教师非常自信时（如 `[0.01, 0.01, 0.97, 0.005, 0.005]`），软标签趋近 one-hot，蒸馏退化为伪 SFT。本项目中豆包的 0.33% 不一致率就导致了此问题。

---

### 10. SFT（Supervised Fine-Tuning, 监督微调）

**定义**：用带有标准答案的数据对预训练模型进行微调。与"预训练"（无监督，预测下一个 token）相对。

**在本项目中的含义**：

| 术语 | 含义 |
|------|------|
| GT SFT | 用标准答案（Ground Truth）做监督微调 |
| Teacher SFT | 用教师预测的答案做监督微调（硬标签蒸馏） |
| CoT SFT | 用教师推理链做监督微调（推理过程 + 答案） |

---

### 11. 梯度累积（Gradient Accumulation）

**原理**：当显存不足以放下大 batch 时，将多个小 mini-batch 的梯度累加后再更新参数，等效于大 batch 训练。

$$\text{有效 batch size} = \text{per\_device\_batch\_size} \times \text{gradient\_accumulation\_steps}$$

本项目中：$2 \times 4 = 8$ 即每 8 个样本更新一次参数。

---

### 12. 学习率调度（Learning Rate Schedule）

**Cosine Annealing（余弦退火）**：学习率按余弦曲线从初始值衰减到接近 0：

$$\eta_t = \eta_{\min} + \frac{1}{2}(\eta_{\max} - \eta_{\min})\left(1 + \cos\left(\frac{t}{T}\pi\right)\right)$$

**Warmup**：训练开始时学习率从 0 线性增长到目标值，避免初始梯度过大导致训练不稳定。

本项目使用 `warmup_ratio=0.1`（前 10% step 做 warmup）+ cosine decay。

---

### 13. BF16 混合精度训练

**BFloat16 (Brain Floating Point 16)**：Google Brain 提出的 16 位浮点格式：
- 1 bit 符号 + 8 bit 指数 + 7 bit 尾数
- 与 FP32 相同的动态范围（8 bit 指数），但精度降低

**vs FP16**：FP16 有 5 bit 指数 + 10 bit 尾数，动态范围小，容易溢出；BF16 更稳定。

**混合精度**：模型权重和激活值用 BF16 存储和计算，梯度更新时用 FP32 精度，兼顾速度和稳定性。显存节省约 50%。

---

### 14. 自一致性（Self-Consistency）与多票采样

**自一致性**（Wang et al., 2023）：对同一问题用温度采样（$T>0$）多次生成答案，取多数一致的答案为最终结果。

**本项目实现**：
- 对每道题采样 $N=9$ 次（1 次 base + 8 次 extra, $T=0.9$）
- 计算一致性分数 = 最高频答案的占比
- 按一致性分级过滤/加权教师标签
- 高一致性（≥0.78）保留全权重；中一致性（0.56~0.78）半权重；低一致性（<0.56）排除

---

### 15. 标签平滑（Label Smoothing）

**定义**：将 one-hot 硬标签"软化"为平滑分布：

$$y_i^{\text{smooth}} = (1-\epsilon) \cdot y_i + \frac{\epsilon}{K}$$

- $\epsilon$：平滑系数（如 0.1）
- $K$：类别数（5）
- 正确类别从 1.0 变为 0.98，每个错误类别从 0 变为 0.005

**本项目教训**：早期 Module 07 中使用 smooth_eps=0.25 导致准确率降至 73.49%（过度平滑）。后改为 0.05 恢复到 79.52%。

**标签平滑 ≠ 真正的软标签**：标签平滑只是机械地分配均匀概率，而教师软标签包含真实的选项间关系信息。

---

### 16. 集成学习（Ensemble）与多数投票

**多数投票**：训练 $K$ 个模型（不同随机种子），每个模型对每道题预测一个答案，取最多票的答案。

$$\hat{y} = \arg\max_c \sum_{k=1}^K \mathbb{1}[f_k(x) = c]$$

**本项目结果**：7-seed 集成投票 = 79.52%，等于最佳单模型，未产生增益。原因是 85.5% 的题目所有模型一致，分歧题投票方向不确定。

---

### 17. 推理链蒸馏（Chain-of-Thought Distillation）

**CoT（Chain-of-Thought）**：Wei 等人（2022）提出的提示策略，让模型"一步步思考"后再给出答案。

**CoT 蒸馏**：收集教师的完整推理链文本，训练学生模型生成同样的推理过程 + 答案。

**本项目失败原因**：
1. 目标序列从 ~5 token 膨胀到 ~200-400 token
2. 训练时学生学生成推理链，评估时只要求输出答案字母→训练/评估不一致
3. 672 条数据不足以让 7B 模型学会可靠的推理范式

---

### 18. 负蒸馏（Negative Distillation）

**定义**：当教师模型能力不足或与学生同级时，蒸馏反而导致学生性能下降的现象。

**本项目案例**：
- Kimi（61.45%）→ 7B：0.00 pp（教师太弱，等于没蒸）
- Qwen2.5-14B（77.11%）→ 7B：**-1.21 pp**（教师=学生级别，引入噪声）

**原因**：弱教师的软标签中错误概率分布误导学生。学生从教师那里学到的"暗知识"实际上是错误知识。

---

### 19. 过拟合（Overfitting）

**定义**：模型在训练集上表现很好，但在测试集上性能退化。

**本项目表现**：
- 训练集准确率通常达到 98%+，但测试集最高 84.34%
- CoT 蒸馏中验证集准确率随训练持续下降：74.32% → 71.62%
- 多 epoch 训练时准确率先升后降

**缓解手段**：
- Stage 1 仅 1 epoch（防止过拟合教师错误）
- LoRA（仅训练 0.35% 参数，天然正则化）
- 早停（按验证集最佳 checkpoint 选模型）

---

### 20. 随机种子（Random Seed）与可复现性

**种子控制的随机性**：
1. 模型参数初始化（LoRA 的 $A$, $B$ 矩阵）
2. 数据打乱顺序（每个 epoch 的样本顺序）
3. Dropout（本项目 dropout=0）

**本项目的种子方差**：
- 7B 实验：std = 2~3 pp（极差最大 9.64 pp）
- 14B 实验：std = 2.55 pp（极差 4.82 pp）
- 每 1 pp 仅对应 83 题测试集中的 0.83 题

**启示**：小测试集上的单 seed 结果不可靠，需多 seed 取均值评估。

---

### 21. 选择性蒸馏（Selective Distillation）

**思想**：并非所有教师标签都有价值。当教师预测与 GT 不一致时，该样本的软标签可能有害。

**本项目策略**：
- Module 14 C1：排除 96 条 `teacher_wrong` 样本，仅用 573 条 `clean_teacher` 训练
- Module 08：按自一致性分数过滤低质量标签

---

### 22. Attention 机制

**Self-Attention**（Vaswani et al., 2017）：Transformer 核心组件：

$$\text{Attention}(Q, K, V) = \text{softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right)V$$

- $Q = XW_Q$（查询），$K = XW_K$（键），$V = XW_V$（值）
- 本项目 LoRA 作用在 $W_Q$ (`q_proj`) 和 $W_V$ (`v_proj`) 上
- 选择 Q 和 V 是因为 Hu et al. (2022) 原论文实验发现这两个投影矩阵的 LoRA 效果最好

---

### 23. Tokenizer 与特殊 Token

**Tokenizer**：将文本转换为 token ID 序列。Qwen2.5 使用基于 BPE 的 tokenizer，词表大小 151,936。

**关键 token**：

| Token | 含义 | 在本项目中的角色 |
|-------|------|----------------|
| A/B/C/D/E | 选项字母 | Choice-Head 提取的 5 个 logit 位置 |
| `<\|im_start\|>` | 对话角色开始 | ChatML 格式标记 |
| `<\|im_end\|>` | 对话角色结束 | ChatML 格式标记 |
| `<think>` | 思考开始 | Qwen3 专用 |
| `</think>` | 思考结束 | Qwen3 non-thinking 需要加 `<think>\n\n</think>` 空前缀 |

---

### 24. Qwen3 Thinking vs Non-Thinking 模式

**Qwen3 双模式架构**：
- **Thinking 模式**（默认）：模型先在 `<think>...</think>` 标签内生成内部推理，再输出最终答案
- **Non-Thinking 模式**：在 assistant 回复开头硬塞 `<think>\n\n</think>\n\n`（空推理块），强制模型跳过思考直接答题

**本项目兼容性问题**：
- 训练时 Qwen3 模型在线评估示 0%（因为默认先生成推理链，max_new_tokens=4 截断了答案）
- 修复：评估时添加 `--qwen3_nothink` 参数，在 prompt 后追加空推理前缀

## 第二部分：预测答辩问答

---

### Q1：请简要介绍你的毕业设计做了什么？

**建议回答**：

我的毕业设计研究基于知识蒸馏的牙科选择题自动答题系统。核心目标是将商业级大型语言模型（如 DeepSeek-V3, 671B 参数）的医学答题能力，迁移到可私有化部署的小型模型（Qwen2.5-7B/14B）上。

我提出了一种**两阶段 Choice-Head 蒸馏框架**：第一阶段利用教师在 A/B/C/D/E 五个选项上的概率分布做 KL 散度蒸馏；第二阶段用标准答案做监督微调校准。通过系统性消融实验（5 种教师、6 种蒸馏方法、15 组实验），最终将 14B 学生模型的准确率从基线 83.13% 提升到 **84.34%**。

---

### Q2：为什么选择知识蒸馏而不是直接用大模型？

**建议回答**：

三个原因：
1. **部署成本**：DeepSeek-V3 有 671B 参数，推理需要多卡集群，按 API 调用计费昂贵。7B 模型单卡即可部署。
2. **数据隐私**：医疗场景对患者数据敏感，不宜将数据送到第三方 API。本地部署小模型可避免数据泄露。
3. **延迟要求**：大模型推理延迟高（秒级），小模型可做到毫秒级响应。

知识蒸馏让我们"一次调用大模型生成标签，终身使用小模型推理"。

---

### Q3：Choice-Head 蒸馏和标准知识蒸馏有什么区别？为什么这样设计？

**建议回答**：

标准 Hinton 知识蒸馏在全词表（15万+维度）上做 KL 散度，要求同时加载教师和学生模型（白盒）。Choice-Head 蒸馏只在 A/B/C/D/E 五个选项 token 上做 KL（5维），教师只需提供 5 个概率值（黑盒）。

设计动机：
1. **任务对齐**：选择题只有 5 个有效候选答案，其余 15 万个 token 的信号是噪声
2. **突破黑盒限制**：DeepSeek-V3 等 API 模型无法获取全词表 logits，但可以通过 logprobs 接口获取选项概率
3. **效率**：显存需求从 ~90GB（白盒，32B+7B）降至 ~22GB（黑盒，仅学生）

实验也验证了 Choice-Head（81.93%）超越了白盒全词表蒸馏（80.72%），说明信噪比比信息量更重要。

---

### Q4：为什么要分两个阶段？一阶段不行吗？

**建议回答**：

两阶段的必要性取决于学生模型的能力：

- **7B 学生需要两阶段**：Stage 1 仅 1 epoch 学习教师分布，但教师有 14% 的标签与 GT 不一致，直接训太久会"学到教师的错误"。Stage 2 用 GT 校准，能纠正偏移。
- **14B 学生只需 Stage 1**：实验发现 14B 的 Stage 1（82.33%均值）优于 Stage 2（80.72%均值），后者反而下降了 1.61pp。这是因为 14B 基线已经很强（83.13%），Stage 2 的 GT SFT 反而"过校准"，丢失了 Stage 1 学到的教师知识。

所以，两阶段是一种灵活框架：根据学生能力决定是否需要 Stage 2。

---

### Q5：你的 α（KL 权重）是怎么选的？为什么是 0.35？

**建议回答**：

通过网格搜索确定。我们测试了 α ∈ {0.02, 0.05, 0.08, 0.15, 0.20, 0.25, 0.30, 0.35, 0.50, 0.70, 0.90}。

关键发现：
- α 过小（<0.1）：蒸馏信号太弱，等于纯 SFT
- α 过大（>0.5）：教师错误主导训练，性能下降
- **α = 0.35 是最优点**：KL 损失和 CE 损失约 2:3 的比例，平衡了教师知识和 GT 锚定

但需要注意，这个最优值与教师质量相关。DeepSeek（14% 不一致率）适合较高 α；如果教师不一致率更高，α 应该更小。

---

### Q6：为什么 DeepSeek 蒸馏效果比豆包好？豆包准确率不是更高吗？

**建议回答**：

这是我们最重要的发现之一。豆包的测试集准确率 98.80% 远高于 DeepSeek 的 87.95%，但蒸馏效果 DeepSeek（+4.82pp）优于豆包（+3.61pp）。

原因是**知识差异性**：
- 豆包与 GT 的不一致率仅 0.33%（604 条中只有 2 条不同），软标签几乎是 one-hot，蒸馏退化为伪 SFT
- DeepSeek 与 GT 的不一致率 14.14%（95/672 条），提供了丰富的"暗知识"——教师在选项之间的犹豫信号

打个比方：如果老师每道题都 100% 确定答案，学生只能学到"正确答案是什么"；但如果老师说"60% 是 A，30% 是 B"，学生还能学到"A 和 B 有相似性"。

---

### Q7：你说教师质量和蒸馏效果是"倒 U 型"关系，能解释一下吗？

**建议回答**：

我们发现教师准确率和蒸馏效果的关系类似倒 U 型曲线：

- **教师太弱**（Kimi 61%）：软标签中错误太多，学生学到噪声，零增益
- **教师与学生同级**（14B 77%）：教师没有超出学生的知识，还引入自身的偏差，导致负蒸馏（-1.21pp）
- **教师适度更强**（DeepSeek 88%）：⭐ 最优区间，提供学生不知道但大部分正确的知识
- **教师过于强大**（豆包 99%）：太自信，软标签退化为 one-hot，暗知识消失

核心指标不是教师的绝对准确率，而是**教师-GT 不一致率**在 5%~15% 的"甜区"。

---

### Q8：LoRA 的 rank 和 alpha 是什么意思？为什么选这个值？

**建议回答**：

LoRA 将权重更新分解为低秩矩阵：$\Delta W = BA$，其中 $B \in \mathbb{R}^{d \times r}$，$A \in \mathbb{R}^{r \times d}$。

- **rank（r=16）**：可训练子空间的维度。越大=表达力越强但参数越多。16 是常用默认值，实验中未做系统搜索（这是研究不足之一）。
- **lora_alpha（α=32）**：缩放因子。实际缩放比例 = α/r = 32/16 = 2.0。相当于放大 LoRA 的更新幅度。

选择 r=16, α=32 基于 Hu et al. (2022) 原论文的推荐和社区最佳实践，在本项目的小数据（672 条）场景下既有足够的表达力又不易过拟合。

---

### Q9：你的测试集只有 83 题，结果可靠吗？

**建议回答**：

坦率地说，83 题的测试集是本研究最大的局限之一。每 1 道题对应 1.2 个百分点，意味着：

- 种子方差 std = 2~3 pp（实际只差 2~3 题）
- 84.34% 和 81.93% 的差距仅 2 题
- 难以做严格的统计显著性检验

我们采取了以下缓解措施：
1. 多种子实验（3~7 seeds）取均值，减少单次随机性
2. 关注结论的一致性：例如"DeepSeek 优于豆包"在多轮实验中重复出现
3. 方法级别的结论（如"多教师融合无增益"）基于多组独立实验，比单一数字可靠

但如果要发表论文，确实需要在更大测试集（如 CMExam 全科）上验证。

---

### Q10：为什么不用全参数微调？LoRA 够用吗？

**建议回答**：

两个原因选择 LoRA：
1. **显存限制**：全参数微调 7B 模型需要 ~60GB 显存（梯度+优化器状态），14B 更是需要多卡；LoRA 只训 0.35% 参数，14B 单卡 H100 即可
2. **小数据防过拟合**：672 条训练数据下，全参数微调极易过拟合；LoRA 的低秩约束天然起到正则化效果

实验证据表明 LoRA 在本任务上是够用的：14B + LoRA 达到 84.34%，接近教师水平（87.95%），说明 LoRA 的表达力不是瓶颈。当然，rank 的最优值未充分探索是一个不足。

---

### Q11：CoT 推理链蒸馏为什么失败了？

**建议回答**：

CoT 蒸馏（Module 12）的准确率仅 73.49%，比基线低 3.62pp。失败原因：

1. **序列长度膨胀**：标准 MCQ 目标只有 ~5 token（答案字母），CoT 推理链有 200-400 token。监督信号被稀释在大量推理文本中。
2. **训练-评估不一致**：训练时学习生成完整推理链，评估时限制 max_new_tokens=4 只取答案字母。模型"学会了推理的格式但没学会正确推理"。
3. **数据不足**：657 条推理链不足以让 7B 模型内化可靠的推理范式。对比 OpenAI 做 CoT 蒸馏用了百万级数据。
4. **验证集持续退化**：74.32% → 71.62%，说明推理模板过拟合严重。

结论：7B 模型 + 小数据场景下，CoT 蒸馏不如直接做选项级蒸馏。直答比思考更高效。

---

### Q12：多教师融合为什么没有带来提升？

**建议回答**：

我们尝试了四教师加权融合（Module 07）和两教师多票过滤（Module 08），都止步于 79.52%。

根本原因是**软标签退化**：融合后 82.4% 的样本是 one-hot 分布（所有教师对这些题一致），只有 17.6% 的样本有非平凡的融合分布。而在这 17.6% 中，融合方向未必更好——因为弱教师（14B, Kimi）的投票反而稀释了强教师（DeepSeek）的信号。

教训是：多教师融合的前提是教师之间有互补的知识，且各自在不同的子集上表现好。但当最强教师在几乎所有题上都更好时，融合只是添加噪声。

---

### Q13：你提到 Stage 2 对强学生有害，怎么理解？

**建议回答**：

这是 Module 13 的关键发现。14B 学生在 Stage 1 后达到 82.33% 均值，但 Stage 2（5 epochs GT SFT）后降到 80.72%，最大回退 4.82pp（seed=11: 84.34% → 79.52%）。

**解释**：Stage 1 让学生从教师那里学到了超越 GT 标签的知识（比如选项间的相对置信度）。Stage 2 的 GT SFT 用 one-hot 硬标签训练，强迫模型"忘记"Stage 1 学到的细粒度分布信息。这就像一个已经理解了深层概念的学生，被迫回去死记硬背标准答案。

**学术术语**：这类似于"灾难性遗忘"（catastrophic forgetting）的一种变体——后续任务（GT SFT）覆盖了前期任务（蒸馏）学到的有价值的表征。

---

### Q14：你的创新点是什么？和现有工作有什么区别？

**建议回答**：

四个创新点：

1. **Choice-Head 蒸馏框架**：现有医学 MCQ 蒸馏大多用全词表白盒或纯文本黑盒方法。我设计的 Choice-Head 方法专门针对选择题结构，在选项级别做 KL 蒸馏。它比白盒信噪比更高（81.93% vs 80.72%），还能使用 API 黑盒教师。

2. **教师质量-蒸馏效果的非线性关系**：通过 5 个教师的控制变量实验，系统性发现了"倒 U 型"效应，这是现有蒸馏文献较少涉及的（大部分工作只用一个教师）。

3. **Stage 2 对强学生有害的发现**：揭示了两阶段训练的适用条件，对蒸馏实践有重要指导意义。

4. **多教师融合的信息瓶颈**：用实验证据否定了"教师越多越好"的直觉，发现融合导致软标签退化的具体机制。

### Q15：如果再给你时间，你会做什么改进？

**建议回答**：

短期可做：
- 扩大测试集：使用 CMExam 完整题库或引入更多牙科题，降低统计方差
- 14B 多种子扫描：当前仅 3 个种子，扩到 10+ 可能发现更优解

### Q16：Qwen2.5 和 Qwen3 有什么区别？为什么 Qwen3 效果不如 Qwen2.5？

**建议回答**：

Qwen3 是通义团队 2025 年推出的新版本，主要特点：
- 原生支持 **Thinking/Non-Thinking 双模式**
- 架构针对推理链（CoT）优化
- 默认行为是先 `<think>` 推理再回答

在本项目的直答场景（non-thinking mode）下，Qwen3-14B 表现不如 Qwen2.5-14B：
- 零样本基线：79.52% vs 83.13%（-3.61pp）
- 蒸馏后最佳：81.93% vs 84.34%（-2.41pp）

原因推测：Qwen3 为推理链优化了架构和预训练数据分布，在被强制跳过思考（non-thinking）时，反而丧失了部分直答能力。就像一个习惯了打草稿的学生，突然被要求不准打草稿直接答题，反而不如一直直接答题的学生。

### Q17：什么是 zero-shot（零样本）？和 few-shot 有什么区别？

**建议回答**：

- **Zero-shot（零样本）**：直接给模型题目让它回答，不提供任何示例。测试模型的"开箱即用"能力
- **Few-shot（少样本）**：在 prompt 中提供几个示例（如 3-5 个带答案的题目），帮助模型理解格式和任务
- **Fine-tuning（微调）**：用训练数据直接更新模型参数

本项目的零样本基线（如 Qwen2.5-7B 77.11%）是 zero-shot 评估。所有的 SFT 和蒸馏训练属于 fine-tuning。

---

### Q18：你用了什么评估指标？为什么只用准确率？

**建议回答**：

评估指标是**精确匹配准确率**（Exact Match Accuracy）：模型输出的答案字母是否与标准答案完全一致。

只用准确率的原因：
1. **任务性质决定**：五选一选择题只有对/错，没有"部分正确"
2. **确定性推理**：评估时 `do_sample=False, temperature=1.0`，每道题只生成一个答案
3. **CMExam 基准惯例**：该数据集的标准评估协议就是使用准确率

如果是多标签或开放式回答，会需要 F1、BLEU 等指标，但选择题场景下准确率是充分且合适的。

### Q19：请解释一下 Transformer 的基本架构。

**建议回答**：

Transformer（Vaswani et al., 2017）基于纯注意力机制：

1. **输入嵌入**：每个 token 转换为向量嵌入 + 位置编码
2. **Multi-Head Self-Attention**：计算序列中每个位置对其他位置的注意力权重
   - $Q = XW_Q, K = XW_K, V = XW_V$
   - $\text{Attention} = \text{softmax}(QK^T/\sqrt{d_k})V$
   - 多头：用不同的 $W_Q, W_K, W_V$ 独立计算，再拼接
3. **Feed-Forward Network (FFN)**：两层线性变换 + 激活函数
4. **残差连接 + LayerNorm**：每个子层都有 $x + \text{SubLayer}(x)$
5. **因果掩码**（Decoder-only 如 Qwen）：只允许关注当前位置之前的 token

Qwen2.5/3 是 Decoder-only Transformer，用 RoPE 位置编码、SwiGLU 激活、GQA（分组查询注意力）等优化。

### Q20：数据集有没有数据泄露风险？模型是不是预训练时就见过这些题？

**建议回答**：

这是一个很好的问题。确实存在这种可能性，因为 CMExam 是公开数据集，Qwen 等模型的预训练语料可能包含部分考试题。

我们的缓解措施：
1. **关注相对提升**而非绝对准确率：蒸馏的价值体现在"比零样本基线提升多少"
2. **零样本基线已固定**：如果模型"记住"了答案，零样本就应该很高（7B 只有 77%），说明记忆不完整
3. **蒸馏后提升说明方法有效**：即使有部分泄露，蒸馏方法仍带来了显著提升

彻底解决需要使用全新未公开的考试题，这超出了本毕业设计的范围。

### Q21：请解释 softmax 函数的作用和性质。

**建议回答**：

Softmax 将任意实数向量转换为概率分布：

$$\sigma(z_i) = \frac{e^{z_i}}{\sum_j e^{z_j}}$$

性质：
- 输出 $\in (0, 1)$，且和为 1
- 保序性：输入越大，输出概率越高
- 指数放大差距：输入的微小差异被指数函数放大

在本项目中：
- **模型输出层**：将 logits 转为选项概率
- **蒸馏**：教师的 softmax 输出就是软标签
- **温度缩放**：$\sigma(z_i/T)$ 控制分布的"尖锐度"
