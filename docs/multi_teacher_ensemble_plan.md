## 方案 1：多老师软标签融合 → 单学生蒸馏（推荐主线）

### 核心思路

在蒸馏阶段就将多个老师的监督信号融合，训练一个学生（一个 LoRA）。

### 融合公式

对每个样本 x，老师 i 的选项概率分布为 p_i(x)，融合后的教师分布：

$$p_{ens}(x) = \sum_{i=1}^{K} w_i \cdot \sigma_i(x) \cdot p_i^{(T)}(x)$$

其中：
- $w_i$：老师 i 的全局权重（按历史准确率归一化）
- $\sigma_i(x)$：老师 i 在样本 x 上的置信度权重（如 max(p_i) 或 entropy-based）
- $T$：温度参数

### 训练损失

$$\mathcal{L} = \alpha \cdot CE(y_{gt},\, s) + \beta \cdot KL(p_{ens}^{(T)} \| s^{(T)})$$

### 推荐超参起点

| 参数 | 起始值 | 搜索范围 |
|------|--------|----------|
| T (温度) | 2.0 | {1.0, 2.0, 3.0, 4.0} |
| α (GT权重) | 0.3 | {0.1, 0.3, 0.5} |
| β (蒸馏权重) | 0.7 | 1 - α |
| 分歧阈值 | 0.6 | 老师一致率 < 阈值时降权 |

### 关键策略

1. **老师能力加权**: 按各老师在 val 集上的准确率做归一化权重
2. **样本置信度加权**: 老师 logits entropy 高 → 降权
3. **分歧样本处理**: 当多数老师不一致时（如 3/5 老师选不同答案），降权或过滤
4. **GT 兜底**: 对所有老师都做错的样本，回退到纯 GT 监督

### 实现步骤

1. 收集所有老师在 train.jsonl 上的软标签（已有各模块的 teacher labels）
2. 新建 `shared/merge_teacher_soft_labels.py`：融合多老师软标签
3. 新建 `06_multi_teacher_ensemble/` 实验目录
4. 训练脚本复用 `shared/train_choice_head_distill.py`，输入融合后的软标签
5. 评估对比独立蒸馏 baseline

### 预期收益

- 单学生部署，推理成本不变
- 利用老师互补性，预期准确率提升 2-5%

## 方案 2：多学生独立蒸馏 → 推理投票集成（上限对照）

### 核心思路

保留现有各老师的独立蒸馏学生 LoRA，推理时同时运行多个学生，投票表决。

### 投票策略

- 多数投票 (majority vote)
- 加权投票（按各学生 val 准确率加权）
- 置信度加权投票（按各学生 logits softmax 最大值加权）

### 实现步骤

1. 各模块独立蒸馏已完成，直接复用现有 LoRA
2. 新建 `shared/ensemble_vote.py`：加载多个 LoRA，逐样本推理+投票
3. 新建 `07_ensemble_vote/` 实验目录，记录评估结果

### 优缺点

- **优点**: 上限可能最高，利用了模型多样性
- **缺点**: 推理成本 ×N，部署复杂度高；不适合生产环境
- **定位**: 作为 upper-bound 实验，评估方案 1 的融合效果是否接近投票天花板

## RAG 兼容性分析

**RAG 与方案 1 更匹配。**

| 维度 | 方案 1 (融合蒸馏) | 方案 2 (投票集成) |
|------|-------------------|-------------------|
| 推理端点 | 单一模型 | 多模型并行 |
| RAG 检索 | 检索一次，拼入一个模型 | 每个学生各自拼入，检索成本×N |
| 上下文一致性 | 单模型统一理解 | 多模型对同一片段理解不一致 |
| 部署复杂度 | 低 | 高 |
| RAG 训练增强 | 可在训练时加入检索片段 | 需为每个学生各自构造 |

### RAG + 方案 1 的典型组合

1. **离线阶段**: 对训练集每个问题做知识检索，将检索片段拼入 prompt 构造增强样本
2. **蒸馏阶段**: 用增强样本让老师生成软标签，再融合蒸馏
3. **推理阶段**: 在线检索 + 单学生推理

## RAG 选型分析

### 推荐: 标准向量 RAG + 元数据过滤

本任务为口腔医学 MCQ，每题考查 1-2 个知识点，答案通常集中在一段教材内容中。
不需要跨多个实体做链式推理，因此 **标准向量 RAG** 即可满足需求。

| 维度 | 标准 RAG (向量检索) | Graph RAG (图谱检索) |
|------|---------------------|----------------------|
| 适用场景 | 单跳事实查询、教科书段落匹配 | 多跳推理、跨实体关系链 |
| 本任务匹配度 | **高** — 题目直接对应教材知识点 | 低 — 无需图谱级多跳推理 |
| 构建成本 | 低: 切分教材 → embedding → 向量库 | 极高: 医学实体抽取+关系标注+图谱维护 |
| 检索质量 | 对"掌握XX知识点"型题目直接命中 | 图谱质量决定上限，医学图谱维护成本大 |
| 与蒸馏结合 | 简单: 检索片段拼入 prompt | 需子图序列化为文本，链路复杂 |

### 元数据增强策略

数据中已有结构化字段可做轻量级预筛选，无需构建完整知识图谱：

- `Disease Group`: 按疾病系统缩小检索范围
- `Clinical Department`: 按科室过滤文档子集
- `Medical Discipline`: 按学科定位教材章节
- `Difficulty level`: 可选，用于采样策略

检索流程: 元数据预筛选 → 向量相似度排序 → Top-K 片段拼入 prompt

### Graph RAG 不推荐的原因

1. 口腔 MCQ 答案通常在单一段落/章节内，不需要跨实体多跳推理
2. 构建高质量医学知识图谱成本极高（实体抽取准确率、关系标注人力）
3. 图谱维护和更新困难，且图谱错误会直接传播到检索结果
4. 元数据预筛选 + 向量检索已能获得类似的"结构化导航"效果

## 创新方向 (论文增强)

### 方向 A: 选择性专家蒸馏 (Teacher Routing) — 创新性 ★★★★★

不同老师在不同科室/疾病类别上准确率不同。训练一个轻量路由器，按样本元数据
自动选择最佳老师或老师组合权重，而非全局统一融合。

$$w_i(x) = \text{Router}(\text{DiseaseGroup}(x),\, \text{Dept}(x),\, \text{Diff}(x))$$

- 数据中已有 `Disease Group`、`Clinical Department`、`Difficulty level` 可直接使用
- 是 MoE 思想在蒸馏端的新应用，现有文献中多老师蒸馏基本用全局固定权重

### 方向 B: 对比蒸馏 (Contrastive Distillation) — 创新性 ★★★★☆

不仅告诉学生"正确选项的概率分布"，还显式利用错误选项的对比信号。

- 从老师的 `Explanation` 字段提取排除逻辑（本数据 829 条全部有 Explanation）
- 构造正例（正确解释）和负例（干扰项为什么错）的对比对
- 损失: $\mathcal{L}_{con} = -\log \frac{\exp(sim(s, e^+)/\tau)}{\sum_j \exp(sim(s, e^j)/\tau)}$
- 标准蒸馏只传递 logits，忽略了 MCQ 中丰富的排除推理信息

### 方向 C: 课程蒸馏 (Curriculum Distillation) — 创新性 ★★★★☆

数据已有 `Difficulty level` 1-5（分布: L1=306, L2=275, L3=157, L4=80, L5=11）。
设计动态课程：根据学生当前能力自适应调整蒸馏难度分布和老师权重。

- 初期: 用最强老师蒸馏简单样本，建立基础
- 中期: 混合难度，引入多老师分歧信号
- 后期: 只蒸馏学生仍做错的难样本，用最擅长该类别的老师

### 方向 D: 自一致性过滤 (Self-Consistency Filtering) — 创新性 ★★★☆☆

对同一题让每个老师做 N 次采样，只保留老师自身一致的标签。

- 5 次中 4 次选 A → 高置信，保留
- 5 次中 2A/2B/1C → 老师不确定，降权或丢弃
- 可构建"老师间一致性 × 老师内一致性"二维置信度矩阵

### 方向 E: 蒸馏-反馈循环 (Distill-then-Probe) — 创新性 ★★★☆☆

第一轮蒸馏后，找出学生仍做错的样本，返回让老师针对性生成更强监督，再做第二轮精细蒸馏。

- Round 1: 标准多老师蒸馏
- Probe: 评估学生，收集错题集
- Round 2: 老师对错题集做 CoT 重新标注，用更强信号蒸馏

### 方向 F: 元数据条件蒸馏 (Metadata-Conditioned Distillation) — 创新性 ★★★☆☆

将元数据作为条件信息注入学生训练：在 prompt 中加入 `[口腔科][难度3][消化系统]`，
或在 LoRA 层面做条件适配。

### 推荐论文组合

Teacher Routing (A) + 课程蒸馏 (C) + 自一致性过滤 (D) → 构成完整框架：
"基于医学领域知识的自适应多老师蒸馏"，每个组件可做消融实验。

### AI 生成数据的潜在风险

由于数据中没有 `source` 字段区分 CMExam 原始 vs AI 生成，无法自动拆分。
AI 生成数据常见问题：

1. **选项不规范**: 选项重复、选项间逻辑不互斥（如上述第 92 行）
2. **答案与解释不一致**: Explanation 论述的结论和 Answer 字段矛盾
3. **知识点错误**: AI 编造不存在的医学事实
4. **难度标注失真**: AI 无法准确评估真实考试难度
5. **分布偏移**: AI 倾向生成某些模式的题目，导致类别不均衡

### 建议清洗策略

1. **添加 source 字段**: 回溯标注每条数据来源（`cmexam` / `ai_generated`），便于后续分析
2. **规则过滤**: 自动剔除选项重复、选项数不足、答案无效的样本
3. **交叉验证**: 用当前最好的老师模型对全部数据做推理，标记老师全错的样本为高风险
4. **人工抽检 AI 子集**: 优先抽检和 CMExam 风格差异大的样本
5. **消融实验**: 分别在"仅 CMExam"和"CMExam+AI"上训练，对比是否有掉分

### Kimi 剔除分析

Kimi 测试集准确率 61.45%，**低于随机猜测的期望上界**（5 选 1 = 20%，但考虑到
部分常识题盲猜率更高，61% 的可信区间仍然很低）。剔除理由：

1. **准确率过低**: 61.45% 远低于其他老师 (最低 77.11%)，权重归一化后贡献极小
2. **噪声风险高**: 在 38.55% 的题上给出错误标签，融合进来会**稀释**高质量老师信号
3. **降权不够**: 即使加权为 0.05，在老师一致的样本上影响不大，但在分歧样本上
   会把投票方向拉偏

**决策: 默认剔除 Kimi，但保留对照实验**

- 主线 (4 老师): Doubao + DeepSeek-V3 + Qwen-32B + Qwen-14B
- 对照 (5 老师): 加入 Kimi，验证弱老师的影响
- 在 Step 2 的消融表中单独报告两组对比

### Step 2: 静态加权融合蒸馏 + LoRA — 骨架 + α/β 网格搜索

**文件夹**: `07_step2_static_fusion/`

**目标**: 建立单学生融合蒸馏 baseline，同时找到最佳 α/β/T 组合

- 输入: 4 老师 (Doubao/DeepSeek/Qwen-32B/Qwen-14B) 在 train 上的软标签
- 默认剔除 Kimi; 保留 5 老师对照组

#### α/β/T 网格搜索方案

固定 β = 1 - α，实际只搜 α 和 T 两个维度:

| α (GT权重) | T (温度) | 含义 |
|-----------|---------|------|
| 0.1 | 1.0 | 重蒸馏、低温(更尖锐) |
| 0.1 | 2.0 | 重蒸馏、中温 |
| 0.1 | 3.0 | 重蒸馏、高温(更平滑) |
| 0.3 | 1.0 | 平衡、低温 |
| **0.3** | **1.5** | **历史最佳起点** |
| 0.3 | 2.0 | 平衡、中温 |
| 0.3 | 3.0 | 平衡、高温 |
| 0.5 | 1.0 | 重GT、低温 |
| 0.5 | 2.0 | 重GT、中温 |
| 0.5 | 3.0 | 重GT、高温 |

**搜索策略**: 两轮
1. **粗搜**: 上述 10 组全跑，找到最佳区域
2. **细搜**: 在最佳区域附近做局部加密 (如 α 在 0.2~0.4 间每 0.05 步长)

**额外消融**:
- 4 老师 vs 5 老师 (含 Kimi)
- 学习率固定 1e-4 (历史最佳)
- 其他超参固定: rank=16, lora_alpha=32, epochs=4, batch=2, grad_acc=8

- 产出: **最佳 (α, T) 下的静态融合 Acc**
- 产出文件: `07_step2_static_fusion/configs/grid_params_best.json`

**Round 1 发现的关键问题**:

1. **假软标签根因**: `prepare_soft_labels.py` 的 `smooth_eps=0.25` 将硬标签转为 (0.80, 0.05, 0.05, 0.05, 0.05)，
   70.1% 样本分布完全相同，**不含真实暗知识 (dark knowledge)**。KL 蒸馏实际优化的是人工标签平滑，而非老师模型的真实不确定性。
2. **T>1 恒劣化**: 因假软标签已近均匀，升温使其更平坦，信号进一步消失。
3. **α 越高越好**: GT 信号是真实的，假软标签贡献有限，所以高 α (重 GT) 效果更好。
4. **早期过拟合**: 最佳 val 普遍在 epoch 1-2，之后下降，缺乏学习率调度器。

#### Round 2 设计 (2026-04-11 细搜/修复)

**修复措施**:
1. smooth_eps 0.25 → 0.05: 教师分布更尖锐 (0.96, 0.01×4)，entropy 0.400 (vs 旧 0.889)
2. 添加 cosine LR scheduler + 10% warmup
3. 增加 best-val-epoch 检查点 + 在最佳 epoch 评测 test
4. 增大搜索空间: α∈{0.5,0.7,0.9,1.0}, lr∈{5e-5,1e-4,2e-4}, T=1.0 固定, epochs=8
