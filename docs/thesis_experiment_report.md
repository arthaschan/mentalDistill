## 1. 研究背景与问题定义

### 1.1 研究动机

牙科医师资格考试题库（CMExam）是评测医学大语言模型能力的重要基准。商业级大模型（如 DeepSeek-V3, 豆包）在此类任务上已达到 87%~99% 的准确率，但其模型参数量巨大、推理成本高昂，不适合在临床终端设备或隐私敏感场景中部署。

**核心问题**：能否通过知识蒸馏，将大型教师模型的答题能力迁移到轻量级学生模型（7B~14B），在保持可部署性的同时最大化准确率？

### 1.2 研究问题

本研究系统性探索以下问题：

1. **教师选择效应**：不同能力水平的教师模型（61%~99%）对蒸馏效果有何影响？
2. **蒸馏方法比较**：白盒 Logit-KL、黑盒 Choice-Head、推理链蒸馏三种范式孰优孰劣？
3. **多教师融合**：多个教师的软标签融合能否超越单教师蒸馏？
4. **学生容量效应**：7B→14B 学生模型升级能否突破准确率天花板？
5. **两阶段训练策略**：蒸馏（Stage 1）+ 监督微调（Stage 2）的级联策略是否有效？

### 3.1 方法 A：纯监督微调（GT SFT）

直接用标准答案监督微调学生模型，不使用任何教师知识。作为所有蒸馏实验的基线。

**损失函数**：

$$\mathcal{L}_{\text{SFT}} = -\log P_\theta(y^* \mid x)$$

其中 $y^*$ 为标准答案，$x$ 为题目输入。

### 3.2 方法 B：白盒 Logit-KL 蒸馏

同时加载教师模型和学生模型，直接最小化两者 logit 分布的 KL 散度。

**损失函数**：

$$\mathcal{L}_{\text{whitebox}} = \alpha \cdot T^2 \cdot D_{\text{KL}}\left(\sigma\left(\frac{\mathbf{z}_T}{T}\right) \Big\| \sigma\left(\frac{\mathbf{z}_S}{T}\right)\right) + (1-\alpha) \cdot \mathcal{L}_{\text{CE}}$$

其中 $\mathbf{z}_T$, $\mathbf{z}_S$ 分别为教师和学生 logit，$T$ 为温度，$\alpha$ 为 KL 权重。

**优势**：获得完整 vocab 级别信息  
**劣势**：需同时加载教师+学生，显存占用巨大（32B+7B ≈ 90GB）

### 3.3 方法 C：黑盒两阶段 Choice-Head 蒸馏（本研究核心方法）

针对选择题特点设计的两阶段蒸馏策略，仅需教师模型的选项级（A/B/C/D/E）概率分布。

**Stage 1 — Choice-Head KL 蒸馏（1 epoch）**：

$$\mathcal{L}_{\text{stage1}} = \alpha \cdot D_{\text{KL}}(\mathbf{p}_T \| \mathbf{p}_S) + (1-\alpha) \cdot \mathcal{L}_{\text{CE}}$$

其中 $\mathbf{p}_T \in \mathbb{R}^5$ 为教师在 A/B/C/D/E 上的概率分布，$\mathbf{p}_S$ 为学生在对应 token 上的 softmax 概率。

**Stage 2 — GT SFT 精校（2~5 epochs）**：

$$\mathcal{L}_{\text{stage2}} = \mathcal{L}_{\text{SFT}}$$

继承 Stage 1 的 LoRA 权重，用标准答案进行监督微调，纠正 Stage 1 中教师错误导致的偏移。

**算法流程**：

```
算法 1: 两阶段 Choice-Head 蒸馏
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
输入: 学生模型 θ_S, 教师软标签 {p_T^(i)}, 训练集 D, GT标签 {y*}
输出: 微调后的学生模型 θ_S'

阶段一：Choice-Head KL 蒸馏
  1. 初始化 LoRA 适配器 Δθ
  2. for each (x, p_T, y*) in D:
       p_S ← softmax(θ_S(x)[A,B,C,D,E])     // 提取5选项概率
       L ← α·KL(p_T ∥ p_S) + (1-α)·CE(y*, p_S)
       Δθ ← Δθ - η·∇L
  3. 保存 Stage1 权重

阶段二：GT SFT 精校
  4. 加载 Stage1 权重 Δθ
  5. for epoch = 1 to E₂:
       for each (x, y*) in D:
         L ← -log P_{θ_S+Δθ}(y* | x)
         Δθ ← Δθ - η·∇L
  6. 返回 θ_S + Δθ
```

### 3.4 方法 D：多教师融合蒸馏

将多个教师的软标签加权融合为统一分布：

$$\mathbf{p}_{\text{fused}} = \sum_{k=1}^{K} w_k \cdot \mathbf{p}_{T_k}, \quad \text{where} \quad w_k = \frac{\text{acc}_k}{\sum_{j=1}^{K}\text{acc}_j}$$

权重按教师测试集准确率归一化。

### 3.5 方法 E：CoT 推理链蒸馏

收集教师模型的完整推理链（Chain-of-Thought），训练学生模型生成推理过程后再输出答案：

$$\mathcal{L}_{\text{CoT}} = -\sum_{t=1}^{|r|} \log P_\theta(r_t \mid x, r_{<t})$$

其中 $r = (r_1, r_2, \ldots, r_{|r|})$ 为教师推理链。

### 6.6 教师特性与蒸馏效果关系分析

```
教师准确率               蒸馏效果
   │                       │
   │  Doubao 98.80% ─────────── +2.41pp (稳定) ─── 教师太强,
   │                       │                       知识增益饱和
   │  DeepSeek 87.95% ────────── +4.82pp (峰值) ── ⭐ 最优区间
   │                       │
   │  Qwen-32B 83.13% ────────── +3.61pp          适中
   │                       │
   │  Qwen-14B 77.11% ────────── -1.21pp          同级噪声教师
   │                       │
   │  Kimi 61.45% ─────────────── ±0.00pp          太弱, 无收益
└──┴───────────────────────┴──
```

**关键发现**：教师质量与蒸馏效果呈**非线性倒 U 型关系**。教师需满足两个条件才能产生正向蒸馏效果：  
1. **能力门槛**：准确率需显著高于学生基线（>83%）  
2. **知识差异性**：教师-GT 不一致率需要适度（5%~15%），提供学生尚未掌握的"暗知识"  

豆包虽有 98.80% 的最高教师准确率，但其 0.33% 的不一致率意味着教师几乎等同于标准答案，蒸馏信号退化为伪标签的 SFT。DeepSeek-V3 以 14.14% 的不一致率提供了最丰富的蒸馏信号。

## 12. 实验九：异构外部教师蒸馏（Llama-3.3-70B）

> **对应模块**：`16_llama70b_choice_head/`  
> **核心发现**：准确率仅 72.8% 的 Llama-70B 外部教师，凭借真实 logprobs 软标签，蒸馏出的 14B 学生与 DeepSeek-V3 教师（87.18%）蒸馏结果竞争性持平。

### 12.1 实验动机

前序所有蒸馏实验的教师模型均来自同一系列（Qwen）或中文优势商业 API（DeepSeek、Doubao）。为验证蒸馏方法的跨架构泛化性，引入一个完全异构的外部教师——**Llama-3.3-70B-Instruct（AWQ INT4 量化版）**。

关键差异：
- **异构架构**：Llama 系列 vs Qwen 系列，不同 tokenizer、不同预训练数据
- **真实 logprobs**：通过 vLLM 本地推理直接获取 ABCDE 五选项上的真实概率分布，无需人工平滑
- **中等准确率**：Llama-70B 在 CMExam 上仅 ~72.8%，远低于 DeepSeek-V3 的 87.18%

### 12.2 教师标签生成

使用 vLLM 0.16.0 加载 AWQ INT4 量化模型（`casperhansen/llama-3.3-70b-instruct-awq`，38GB），
通过 `SamplingParams(max_tokens=1, temperature=0, logprobs=20)` 提取真实概率分布。

**Llama tokenizer ABCDE token IDs**：A=32, B=33, C=34, D=35, E=36

> **发现 F15**：**教师准确率远非决定性因素——真实 logprobs 的价值超越准确率差距**。Llama-70B 教师准确率仅 72.79%（低于 DeepSeek-V3 的 87.18% 达 14.39pp），但蒸馏后学生全量准确率仅低 1.42pp（87.25% vs 88.67%）。在牙科子集上甚至反超（80.00% vs 79.20%）。这说明教师的概率分布信息（"哪些选项容易混淆"）比其绝对正确率更重要。

> **发现 F16**：**真实 logprobs 无需人工平滑即可有效蒸馏**。DeepSeek-V3 的 API 只返回 one-hot 标签，需要 smooth_eps=0.25 人工注入不确定性。Llama-70B 通过 vLLM 直接提取真实概率分布（平均熵 0.1769），使用 smooth_eps=0.0 即可获得竞争性蒸馏效果。

> **发现 F17**：**异构教师蒸馏可行**。Llama 和 Qwen 使用完全不同的 tokenizer、预训练数据和架构，但 Choice-Head 蒸馏框架仅依赖选项级概率分布（5 维向量），天然跨架构兼容。这验证了 Choice-Head 方法的通用性。

## 15. 关键发现与创新点

### 15.1 创新点

**创新点 1：两阶段 Choice-Head 蒸馏框架**

针对选择题场景设计的专用蒸馏方法，仅利用教师模型在 5 个选项上的概率分布（而非全 vocab logit），解决了黑盒 API 教师无法进行白盒蒸馏的瓶颈。两阶段策略（先蒸馏学知识 → 再 SFT 校准）兼顾了知识迁移与答案准确性。

**创新点 2：教师质量-蒸馏效果的非线性关系发现**

通过 5 个不同能力水平的教师进行控制变量实验，首次系统性揭示了教师准确率与蒸馏增益之间的"倒 U 型"关系。发现教师-GT 不一致率（14%）比教师绝对准确率（87% vs 99%）更能预测蒸馏效果。

**创新点 3：Stage 2 对强学生有害的发现**

在 14B 学生实验中发现，当学生模型基线已足够强（≥83%）时，Stage 2 的 GT SFT 反而导致性能回退（-1.61 pp 均值，最大 -4.82 pp）。这揭示了蒸馏训练中"过校准"的风险，对实际部署有重要指导意义。

**创新点 4：多教师融合的信息瓶颈发现**

通过四教师融合实验和多票自一致性过滤实验证明：当教师质量足够高时，融合软标签退化为 one-hot 分布（82.4% 样本），多教师的"集体智慧"不产生额外收益。这从实证角度否定了"教师越多越好"的直觉。

### 14.3 学生模型超越教师模型的场景分析

在全部实验中，学生模型超越教师模型的情况出现在以下两类场景：

#### 场景一：弱教师被学生天然超越（Module 04 — 反面案例）

| 项目 | 值 |
|------|-----|
| 教师模型 | Kimi (moonshot-v1-32k) |
| 教师准确率 | 61.45%（83 题牙科测试集） |
| 学生模型 | Qwen2.5-7B-Instruct |
| 学生准确率 | 77.11% |
| 超越幅度 | **+15.66 pp** |
| 分析 | 教师准确率（61.45%）低于学生的纯 GT SFT 基线（77.11%），蒸馏实际无正向增益。学生的准确率完全来自 Stage 2 的 GT SFT 校准，与教师标签无关。这说明弱教师的软标签提供的是噪声而非知识，蒸馏退化为纯 SFT。 |
| 意义 | **负面案例**——证明教师准确率存在下限（≥70%），低于此阈值蒸馏无实际价值。 |

#### 场景二：强学生 + 大数据蒸馏真正超越强教师（Module 15 — 核心发现）

| 项目 | 值 |
|------|-----|
| 教师模型 | DeepSeek-V3 (671B MoE) |
| 教师准确率 | 87.18%（991 题全量测试集） |
| 学生模型 | Qwen2.5-14B-Instruct |
| 学生准确率（均值） | **88.67% ± 0.31pp**（3 seed 均值） |
| 学生准确率（最佳） | **89.10%**（seed=8） |
| 超越幅度（均值） | **+1.49 pp** |
| 超越幅度（最佳） | **+1.92 pp** |
| 超越一致性 | **3/3 seed 全部超越教师**（88.40%, 88.50%, 89.10%） |

**为什么学生能超越教师？**

1. **知识蒸馏的"双通道学习"效应**：Stage 1 的 Choice-Head KL 损失同时包含两个信号——教师的软标签分布和 GT 硬标签的交叉熵。学生模型从教师的"犹豫模式"中学到选项间的相对难度，同时又受 GT 标签约束避免继承教师的错误。当 $\alpha=0.35$ 时，GT 信号权重（65%）足以纠正教师的 12.82% 错误预测。
2. **数据规模优势**：4608 题全学科训练数据（覆盖 7 个医学学科）远大于牙科 672 题，学生模型获得了更丰富的跨领域知识表征。而教师的错误率在不同学科间分布不均，学生可以在教师弱项学科上"以量取胜"。
3. **参数效率的结构化优势**：DeepSeek-V3 是 671B 参数的稀疏 MoE 架构，每次推理仅激活约 37B 参数；而 Qwen2.5-14B 是 14.8B 参数的稠密架构，所有参数全部参与推理。稠密架构在小规模知识域（如医学 MCQ）上的参数利用率可能更高。
4. **LoRA 微调的正则化效应**：LoRA 的低秩约束（rank=16）天然具有正则化作用，防止对教师错误样本的过拟合。

**注意事项**：

- 在牙科子集（125 题）上，14B 学生均值恰好等于教师（79.20% vs 79.20%），仅 seed=42 微幅超越（80.00%），说明在教师已达天花板的细分领域，学生的超越空间极其有限。
- 7B 学生在全量测试集上的均值为 85.60%，始终低于教师的 87.18%（-1.58pp），说明 7B 容量是超越教师的硬约束。
- 在旧实验（83 题牙科测试集）上，**没有任何学生模型真正超越其强教师**（Kimi 除外），14B 最佳 84.34% 仍低于 DeepSeek 87.95%。

#### 超越条件总结

| 必要条件 | 本实验验证 |
|---------|-----------|
| 学生模型容量足够大（≥14B） | ✓ 7B 从未超越强教师 |
| 训练数据量足够大（≥4000+） | ✓ 672 题训练时 14B 最高 84.34%，4608 题训练时提升至 88.67% |
| 教师准确率处于"最优蒸馏区间"（80%~90%） | ✓ DeepSeek 87.18% 有足够错误供学生修正 |
| 采用合适的蒸馏策略（仅 Stage 1，不加 Stage 2） | ✓ Stage 2 对强学生有害 |

### 14.5 Qwen3-14B 实验失败分析

#### 背景

Module 14 尝试用 Qwen3-14B 替代 Qwen2.5-14B 作为学生模型，配合 DeepSeek-V3 教师标签进行 Choice-Head 蒸馏，以验证更新一代的模型架构是否能带来更好的蒸馏效果。

#### 实验结果

| 阶段 | 方法 | 准确率（牙科 83 题） |
|------|------|---------------------|
| Phase 0 | Qwen3-14B 零样本（non-thinking） | 79.52%（66/83） |
| Phase 1 (C1) | Choice-Head 蒸馏 × 3 seeds | 训练时报告 **0.00%**（实际 80.72% 均值） |
| Phase 2 (C3) | CoT 推理链 SFT × 2 seeds | 69.88% ~ 74.70%（退化） |

对比 Qwen2.5-14B：零样本 83.13% → 蒸馏后 84.34%（均优于 Qwen3-14B）。

#### 失败原因一：Thinking 模式的评估 Bug

Qwen3 模型默认的生成行为是先输出 `<think>` 推理链，然后再输出最终答案。训练脚本 `train_choice_head_distill.py` 中的 `evaluate_generation` 函数使用 `max_new_tokens=4` 做在线评估，但**没有添加 `<think>\n\n</think>\n\n` 前缀来禁用思考模式**。

结果是：模型生成的前 4 个 token 始终是 `<think>\n` 等思考标记，**永远无法输出答案字母**，导致训练过程中验证准确率全部显示为 0%，best checkpoint 选择机制失效。

> 训练本身完全正常——loss 从 ~2.8 下降到 ~0.04，权重更新正确。事后用 `--qwen3_nothink` 参数重新评估，C1 实际准确率为 79.52%~81.93%。

#### 失败原因二：Qwen3 的 non-thinking 模式天然弱于 Qwen2.5

| 对比维度 | Qwen3-14B | Qwen2.5-14B-Instruct |
|---------|-----------|---------------------|
| 零样本准确率（83 题） | 79.52% | **83.13%**（-3.61pp） |
| 蒸馏后最佳（83 题） | 81.93% | **84.34%**（-2.41pp） |
| 架构层数 | 40 层 | **48 层**（更深） |
| FFN 中间维度 | **17408**（更宽） | 13824 |
| 设计目标 | 推理链（thinking）优化 | 指令跟随（直答）优化 |

**根本原因分析**：

1. **架构适配性错配**：Qwen3 采用"更宽更浅"的架构（40 层 × 17408 FFN），专为 Chain-of-Thought 推理优化。当被强制进入 non-thinking 模式（通过 `<think>\n\n</think>\n\n` 前缀跳过推理）时，模型丧失了其设计优势，相当于"一个习惯打草稿的学生被要求不打草稿直接答题"。
2. **预训练数据分布偏移**：Qwen3 使用约 36T token 预训练，重点在数学和代码等推理型任务。直答型的医学 MCQ 不属于其预训练的优势领域，预训练知识表征中医学领域的"直觉响应"不如 Qwen2.5。
3. **指令微调策略差异**：Qwen2.5-14B-Instruct 经过大量直答型指令微调，对 "直接输出答案字母" 这一模式有良好的适配；Qwen3 的后训练则以推理链为主，直答能力是"副产品"而非设计目标。

#### 为什么不能用 Qwen3 的 thinking 模式做选择题？

理论上，Qwen3 可以在 thinking 模式下先推理再答题，但存在以下实际障碍：

| 障碍 | 说明 |
|------|------|
| **Choice-Head 蒸馏不兼容** | 本研究的核心方法（Choice-Head KL）直接对比学生与教师在 A/B/C/D/E 五个 token 上的概率分布。如果允许 thinking 模式，模型会先生成数百个推理 token，答案字母出现在不确定位置，无法对齐用于 KL 计算。 |
| **训练-推理不一致** | 若训练时使用 non-thinking 强制直答，推理时使用 thinking 模式，两种模式下模型的内部表征和输出分布可能完全不同，训练收益无法迁移。 |
| **CoT SFT 已验证失败** | Module 12（7B）和 Module 14 Phase 2（Qwen3）都尝试了 CoT 推理链 SFT，结果均严重退化（-3.62pp 和 -9.64pp）。原因是 657 条训练数据不足以让模型学会可靠的医学推理链，监督信号被稀释在冗长的推理步骤中。 |
| **推理开销巨大** | thinking 模式下每道题需生成数百个 token 的推理链，推理时间增加 10~50 倍，不满足实际部署需求。 |

#### 结论

Qwen3-14B 在本任务中不如 Qwen2.5-14B-Instruct，不是因为 Qwen3"不能"做选择题，而是因为：

1. **现有蒸馏框架（Choice-Head）要求直答模式**，而 Qwen3 的直答能力天然弱于 Qwen2.5；
2. **在 non-thinking 模式下**，Qwen3 的零样本基线就低 3.61pp，蒸馏无法弥补这一差距；
3. **让 Qwen3 使用 thinking 模式**则需要重新设计蒸馏方法（如 CoT 蒸馏），而本研究已证明在小数据量下 CoT 蒸馏效果为负。

## 15. 研究不足

### 15.1 数据规模限制

- **~~测试集仅 83 题~~**（已在 Module 15 中解决）：原测试集每题对应 1.2 个百分点，种子方差大。Module 15 扩展到 991 题（±2.5pp CI），种子极差从 4.82pp 降至 0.70pp
- **~~训练集仅 672 题~~**（已在 Module 15 中解决）：扩展到 4608 题全量训练，7B 从 79.52% 天花板提升到 85.60%
- **单一科目局限已缓解**：Module 15 使用全量 7 学科数据训练和测试，同时保留牙科子集对比

### 15.2 教师选择偏限

- 仅测试了 5 个教师模型，缺乏更多中间能力水平（如 85%、90%、95%）的教师进行精细粒度分析
- 豆包的超高准确率（98.80%）可能是特定于牙科领域的异常值，不代表通用能力

### 15.3 方法局限

- **Choice-Head 仅适用于选择题**：无法直接迁移到开放式问答、病例分析等场景
- **未探索 LoRA 秩的影响**：固定 rank=16，未系统探索 rank∈{4,8,16,32,64} 的效果
- **未使用课程学习**：所有样本权重相同，未按难度排序训练
- **未使用数据增强**：仅使用原始 CMExam 数据，未尝试同义改写、选项打乱等增强手段（Module 02 的数据增强实验仅在无蒸馏 SFT 上进行）

### 15.4 评估局限

- **确定性推理**：仅评估 do_sample=False 下的表现，未考虑采样多样性
- **Qwen3 评估兼容性**：训练过程中的在线评估未添加 `<think>\n\n</think>` 前缀，导致 Qwen3 模型在训练时显示 0% 准确率（需离线重新评估）
- **未进行人工盲评**：所有评估基于精确匹配，未考虑模型推理过程的质量
